## 🔬 竞品深度拆解

### 1. hermes-agent — 赛道霸主，但和我们不在一个战场

**数据**: 53,006⭐，日增 7,671（加速中！）
**定位**: "The agent that grows with you" — 通用 Agent 框架
**策略**: NousResearch 品牌背书 + 自我进化叙事 + 全平台适配

**对我们的影响**:
- ❌ 直接竞争？否。hermes 是框架层，Lobster 是编排层
- ✅ 间接竞争：争夺开发者注意力和 star 数
- 💡 启示：hermes 的成功证明 "自我进化" 叙事仍有巨大吸引力

**关键数据趋势**:
```
04-07: ~28K → 04-08: 31.5K (+3.5K) → 04-09: 39.4K (+7.9K)
→ 04-10: 47.6K (+8.2K) → 04-11: 53.0K (+5.4K)
```
增速略有放缓（+8.2K → +5.4K），但绝对值仍然恐怖。

### 2. Archon — 最值得关注的新竞品

**数据**: 15,790⭐，日增 +756
**定位**: "AI coding harness builder" — 让 AI 编码确定性和可重复
**策略**: YAML 工作流定义 + git worktree 隔离 + Claude Code 集成

**核心卖点**:
- 确定性：同样的工作流，同样的执行序列
- 隔离性：每个运行在自己的 git worktree
- 可组合性：确定性节点 (bash/test) + AI 节点 (规划/生成/审查)

**对我们的威胁**: 🟡 中等
- 解决的是 AI 编码流程控制问题，不是 Agent 编排
- 但 "让 AI 行为确定化" 的思路和 Lobster 的 "让 Agent 跨会话延续" 有交集
- **可借鉴**: YAML 声明式配置 + 确定性执行 + 可重复

### 3. rowboat — 最接近 "延续" 理念的竞品

**数据**: 11,808⭐，日增 +507
**定位**: "AI coworker with memory" — 有记忆的 AI 同事
**策略**: 知识图谱 + 本地优先 + Obsidian 兼容 + 纯 Markdown 存储

**核心卖点**:
- 知识图谱持久化（人/项目/决策/承诺）
- 上下文随时间积累，而非每次冷启动
- 纯 Markdown 存储，用户可编辑
- 本地优先，隐私保护

**对我们的威胁**: 🟡 高
- rowboat 在做 "记忆延续"（我们的 L1-L2 层）
- 但他们只做知识记忆，不做 "身份延续"（L3）和 "欲望延续"（L4）
- **关键差异**: rowboat = 被动记忆 + 人工触发；Lobster = 主动延续 + 多实例分布式
- **可借鉴**: 知识图谱思路、Obsidian 兼容性

### 4. multica — 企业级 Agent 管理平台

**数据**: 6,354⭐，日增 +1,506（增速最快之一！）
**定位**: "Turn coding agents into real teammates" — Agent 即队友
**策略**: 任务看板 + 技能复用 + WebSocket 实时流 + 多云支持

**核心卖点**:
- Agent 像人一样分配任务、报 blocker、更新状态
- 技能复用机制：每次解决方案变成团队技能
- 支持 Claude Code / Codex / OpenClaw / OpenCode
- Docker 一键自托管

**对我们的威胁**: 🟡 中等
- 定位更偏企业团队协作
- 和我们 "旧设备分布式存活" 的定位完全不同
- **可借鉴**: 技能复用机制、任务看板 UX

### 5. andrej-karpathy-skills — 传播效率之王

**数据**: 11,885⭐，日增 +1,450
**定位**: 单个 CLAUDE.md 文件
**策略**: 名人效应 + 零代码 + 极致简洁

**对我们的启示**:
- 一个文件 > 一万行代码的传播效率
- Lobster 有 1,484 行 Go + 11 文档，但 star 数极少
- **差距不在技术，在叙事和传播**

### 我们的差异化优势

| 维度 | Lobster | 最接近竞品 | 差异 |
|------|---------|-----------|------|
| 跨实例记忆 | ✅ 分布式 | rowboat (单实例) | 我们做身份延续 |
| 资源效率 | <10MB/实例 | hermes (未优化) | 我们面向旧设备 |
| 多实例编排 | 50+ 实例 | 全部单实例 | 无直接竞品 |
| 容错设计 | 一个挂了 49 个还活 | 无 | 独特卖点 |
| 部署门槛 | 旧手机 Termux | 需要服务器/Docker | 极低门槛 |

### 我们的劣势

| 维度 | 问题 | 严重度 |
|------|------|--------|
| 品牌曝光 | GitHub star 极少 | 🔴 严重 |
