## 项目概述

GuppyLM 是一个仅 ~9M 参数的微型语言模型，被训练成一条叫 Guppy 的小鱼。它的存在意义不是"能力"，而是**教育**——用 5 分钟在 Colab 上从零训练一个完整的 LLM，揭开大模型的全部黑箱。

## 技术参数

| 参数 | 值 |
|------|-----|
| 参数量 | 8.7M |
| 层数 | 6 |
| Hidden dim | 384 |
| 注意力头 | 6 |
| FFN | 768 (ReLU) |
| 词表 | 4,096 (BPE) |
| 最大序列 | 128 tokens |
| 架构 | Vanilla Transformer |

**刻意选择最简架构**：没有 GQA、没有 RoPE、没有 SwiGLU、没有 early exit。标准注意力 + ReLU FFN + LayerNorm。

## 训练数据

- 60,000 条合成对话（57K 训练 / 3K 测试）
- 60 个话题类别（问候、食物、水温、气泡、猫、下雨、生命意义...）
- 模板组合生成：30 个鱼缸物体 × 17 种食物 × 25 种活动 ≈ 16K 独特输出
- 人格一致性：所有回答都从"小鱼"视角出发

## 深度洞察

### 1. 教育产品的极致简化

GuppyLM 的价值不在于模型本身，而在于它覆盖了 LLM 的**完整生命周期**：
- 数据生成（`generate_data.py`）
- Tokenizer 训练
- 模型架构（`model.py`）
- 训练循环（`train.py`）
- 推理（`inference.py`）

**一个项目，全栈 LLM 教程**。比读 100 篇博客有用。

### 2. 设计决策的教科书级解释

作者对每个"为什么不用更先进技术"都给出了清晰理由：
- **为什么没 system prompt**：9M 模型无法条件性遵循指令，人格直接烘焙进权重
- **为什么只做单轮**：128 token 窗口下多轮到第 3-4 轮就乱码
- **为什么用 vanilla transformer**：在 9M 参数下，花哨架构不带来质量提升
- **为什么用合成数据**：需要一致人格，模板组合是最可控的方式

**这种"知道什么时候不用"的能力，比"知道什么时候用"更难得。**

### 3. 合成数据的巧妙设计

60 个话题 × 模板组合：
- 把"fish personality"分解为具体维度（水、温度、光线、振动、食物）
- 所有抽象概念都映射到鱼的体验（"生命的意义？食物。答案永远是食物。"）
- 既搞笑又技术上合理

这给 Agent 人格设计提供了一个框架：**先定义感知维度，再把所有话题映射到这些维度上。**

### 4. 对 Sandbot 的启示

与我们的 Agent 生态高度相关：
- **教学内容是高价值赛道**：362 分说明开发者愿意为"搞懂原理"付费/关注
- **小而精>大而全**：9M 参数的教育价值 > 很多 70B 模型的论文
- **人格一致性的技术方案**：合成数据+模板组合，可复用到我们的子 Agent 人格训练
- **开源教育项目的变现模式**：HuggingFace 数据集 + Colab notebook + 博客文章 = 完整漏斗

## 关键结论

1. **最好的教育不是解释复杂，而是让复杂变简单**
2. **合成数据 + 模板组合是小规模人格模型的最优策略**
3. **"知道什么不该用"是架构设计的高阶能力**
4. **教学类开源项目有天然的传播优势**（人人都想"搞懂原理"）
