## 测试结果

### 1. 单次调用基线
| 指标 | 值 |
|------|-----|
| 响应时间 | 14.39s |

| 状态 | ✅ 正常 |

### 2. 连续调用上限
| 指标 | 值 |
|------|-----|
| 测试次数 | 30 次 |
| 成功次数 | **30/30 (100%)** |
| 平均响应时间 | 4.13s |
| 平均 token | 192 tokens/调用 |
| 总 token | 5,775 |
| 状态 | ✅ **调用次数无上限** |

### 3. 并发调用上限
| 并发数 | 结果 | 说明 |
|--------|------|------|
| 2 | ✅ 2/2 | 正常 |
| 5 | ❌ 4/5 | **第 5 个请求返回 429** |

**错误信息**:
```
HTTP 429: "concurrency allocated quota exceeded. please try again later."
```

**结论**: 并发限制为 **4 路**

### 4. Token 消耗速率测试
| 指标 | 值 |
|------|-----|
| 发送请求数 | 5 |
| 成功 | 2/5 |
| 失败 | 3/5 (429 并发超限) |
| 总 token (成功) | 7,941 |
| 单次最大 token | 5,979 (completion 5,957) |

**失败原因**: 5 路并发触发 429，不是 token 超限

## 核心结论

| 限制类型 | 是否有限制 | 限制值 |
|----------|-----------|--------|
| 每分钟调用次数 | ❌ 无 | 不限 |
| 每日调用次数 | ❌ 无 | 不限 |
| **并发请求数** | ✅ **有** | **4 路** |
| 单次最大 token | ❌ 未触发 | 至少 5,957 output tokens |
| 每分钟总 token | ❌ 未触发 | 未知 |

### 关键发现
1. **不是调用次数超限** — 30 次连续调用 100% 成功
2. **不是 token 超限** — 单次输出 5,957 tokens 正常
3. **唯一限制是并发数 = 4** — 同时发起第 5 个请求就 429

## 对 Sandbot 的影响

### 不会触发超限的场景
- ✅ Cron 定时任务（顺序执行，每次 1 个）
- ✅ 单条用户对话
- ✅ 串行调用子 Agent

### 会触发超限的场景
- ❌ 同时 spawn 超过 4 个子 Agent
- ❌ 用户短时间内发送多条消息
- ❌ 多个 Cron 任务同时运行

### 安全策略
- 并发上限: **≤4 路**（当前配置 10 次/分钟太宽松，建议改为 4）
- 子 Agent spawn: 串行化，不要并发超过 4 个
- 请求失败后: 等待 30-60 秒重试
