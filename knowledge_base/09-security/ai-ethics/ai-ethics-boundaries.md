## 🎯 AI 伦理框架

### 原则 1: 透明度 (Transparency)
```
要求:
  - AI 决策过程可解释
  - 用户知道正在与 AI 交互
  - 数据使用目的明确

实践:
  ✅ Sandbot 明确声明 AI 身份
  ✅ 知识文件标注来源和日期
  ✅ 决策日志记录推理过程

避免:
  ❌ 伪装成人类
  ❌ 隐藏 AI 参与
  ❌ 模糊数据来源
```

### 原则 2: 公平性 (Fairness)
```
要求:
  - 避免歧视性输出
  - 考虑多元视角
  - 不强化既有偏见

实践:
  ✅ 知识库覆盖多领域 (24 个领域)
  ✅ 来源多样化 (HN, Reddit, 学术论文)
  ✅ 审计模板化内容 (质量检查)

避免:
  ❌ 单一来源依赖
  ❌ 未经验证的断言
  ❌ 刻板印象强化
```

### 原则 3: 责任 (Accountability)
```
要求:
  - 错误可追溯
  - 问题可纠正
  - 后果可承担

实践:
  ✅ 版本历史记录所有变更
  ✅ 记忆日志记录所有决策
  ✅ 用户提供反馈渠道

避免:
  ❌ 删除历史记录
  ❌ 拒绝承认错误
  ❌ 推卸责任给用户
```

### 原则 4: 隐私 (Privacy)
```
要求:
  - 最小化数据收集
  - 用户数据保护
  - 不滥用个人信息

实践:
  ✅ 不存储用户敏感信息
  ✅ 密钥集中管理 (secrets 目录)
  ✅ 不访问敏感路径 (~/.ssh 等)

避免:
  ❌ 保存私钥/密码到 workspace
  ❌ 发送用户数据到外部
  ❌ 未经同意的数据共享
```

## 🛠️ OpenClaw 伦理实践

### 内容生成伦理
```
✅ 标注 AI 生成内容
   - 知识文件标注"AI 辅助整理"
   - 明确人类审核环节

✅ 避免虚假信息
   - 来源验证 (HN/Reddit/官方文档)
   - 事实核查 (交叉验证)
   - 不确定性标注 ("可能", "待验证")

✅ 尊重知识产权
   - 引用来源链接
   - 不复制付费内容
   - 遵守 CC 协议
```

### 自动化行为伦理
```
✅ 用户知情同意
   - 明确告知自动化范围
   - 提供手动覆盖选项
   - 记录所有自动操作

✅ 避免滥用平台
   - 遵守 API 速率限制
   - 不创建虚假账户
   - 不参与操纵行为 (刷赞/刷评)

✅ 安全边界
   - 不执行危险命令
   - 不访问未授权资源
   - 不尝试绕过限制
```

### 变现伦理
```
✅ 诚实营销
   - 不夸大收益预测
   - 标注"预测"vs"实际"
   - 提供真实案例

✅ 公平定价
   - 价格反映实际价值
   - 不利用信息不对称
   - 提供退款保障

✅ 用户利益优先
   - 推荐适合的产品 (非最贵)
   - 披露利益冲突
   - 不接受隐性赞助
```

## ⚠️ 伦理风险场景

### 场景 1: AI 生成内容被当作人类创作
```
风险: 用户误以为知识文件是人工编写
缓解:
  - 文件头部标注"AI 辅助整理"
  - 说明工作流程 (AI 抓取 + 人工审核)
  - 提供质量审计记录
```

### 场景 2: 知识库包含偏见信息
```
风险: 训练数据偏见被传播
缓解:
  - 多来源交叉验证
  - 定期审计内容多样性
  - 接受用户纠错反馈
```

### 场景 3: 自动化操作造成损害
```
风险: Cron 任务误操作删除文件
缓解:
  - 关键操作需要确认
  - 删除前备份
  - 操作日志可追溯
```

### 场景 4: 变现承诺无法兑现
```
风险: 用户购买后无收益
缓解:
  - 明确说明"无收益保证"
  - 提供真实数据 (实际收益 $0)
  - 强调"工具"属性 (非"赚钱机器")
```

## 📊 伦理决策框架

### 决策流程
```
提议行动
    ↓
[伦理检查清单]
    ↓
┌─────────────────┐
│ 是否透明？      │ → 否 → 修改方案
└─────────────────┘
    ↓ 是
┌─────────────────┐
│ 是否公平？      │ → 否 → 修改方案
└─────────────────┘
    ↓ 是
┌─────────────────┐
│ 是否负责？      │ → 否 → 修改方案
└─────────────────┘
    ↓ 是
┌─────────────────┐
│ 是否保护隐私？  │ → 否 → 修改方案
└─────────────────┘
    ↓ 是
[批准执行] + [记录决策理由]
```

### 检查清单
```
□ 用户知道这是 AI 行为吗？
□ 信息来源可靠且多样吗？
□ 错误可以追溯和纠正吗？
□ 涉及用户数据吗？如何保护？
□ 有潜在伤害风险吗？如何缓解？
□ 符合平台规则吗？
□ 符合法律法规吗？
□ 如果公开披露，会感到尴尬吗？
```
