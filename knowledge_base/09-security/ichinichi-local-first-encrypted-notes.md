## 🔬 技术特性

### 核心设计原则
```
1. Local-First (本地优先)
   - 数据存储在本地设备
   - 云端仅作为同步备份
   - 离线可用
   - 用户完全控制

2. End-to-End Encrypted (端到端加密)
   - 加密在客户端完成
   - 服务器无法读取内容
   - 密钥由用户保管
   - 零知识架构

3. One Note Per Day (每日一记)
   - 限制每日笔记数量
   - 鼓励深度思考
   - 减少决策疲劳
   - 形成日记习惯
```

### 技术架构
```
┌─────────────────────────────────────┐
│         Client (Browser/App)        │
│  ┌─────────────────────────────┐   │
│  │  Note Editor                │   │
│  │  Encryption (Client-side)   │   │
│  │  Key Management             │   │
│  └─────────────────────────────┘   │
└─────────────────────────────────────┘
              ↓ (encrypted)
┌─────────────────────────────────────┐
│         Sync Server                 │
│  - 存储加密数据                     │
│  - 无法解密内容                     │
│  - 仅负责同步                       │
│  - 零知识证明                       │
└─────────────────────────────────────┘
```

### 加密方案
| 组件 | 算法 | 用途 |
|------|------|------|
| 数据加密 | AES-256-GCM | 笔记内容 |
| 密钥派生 | PBKDF2/Argon2 | 密码→密钥 |
| 密钥交换 | X25519 | 多设备同步 |
| 签名 | Ed25519 | 完整性验证 |

## 💡 核心价值

### 1. 隐私保护
```
传统云笔记风险:
  - 服务商可读取内容
  - 数据泄露风险
  - 政府调取可能
  - 广告定向分析

Ichinichi 方案:
  - 端到端加密
  - 服务商零知识
  - 泄露也无风险
  - 无广告追踪
```

### 2. 数据主权
```
用户控制:
  - 数据存储在本地
  - 可导出完整数据
  - 可自托管同步
  - 可删除不留痕

对比传统服务:
  - Notion: 数据锁定
  - Evernote: 导出困难
  - Google Keep: 依赖 Google
```

### 3. 心理健康
```
"每日一记"理念:
  - 限制数量 → 减少焦虑
  - 固定格式 → 降低决策成本
  - 深度思考 → 提高质量
  - 持续记录 → 形成习惯

心理学依据:
  - 日记疗法 (Journaling Therapy)
  - 正念练习 (Mindfulness)
  - 反思习惯 (Reflection)
```
