## 📊 核心事实

### 攻击原理
- **名称**: Glassworm (玻璃虫)
- **类型**: Unicode 同形字符攻击 (Homoglyph Attack)
- **机制**: 用视觉相似的 Unicode 字符替换 ASCII 字符
- **目标**: 开发者视觉欺骗 + AI 代码生成污染

### 2026 年新特点
| 特点 | 2024 原始版本 | 2026 回归版本 |
|------|--------------|--------------|
| 传播渠道 | GitHub 手动提交 | AI 代码生成推荐 |
| 影响范围 | 单个仓库 | npm/PyPI 供应链 |
| 检测难度 | 中等 (肉眼可辨) | 高 (AI 推荐背书) |
| 攻击规模 | 数十个仓库 | 数百个仓库 |

### 攻击示例
```javascript
// 正常代码
const apiKey = "sk-abc123";

// Glassworm 攻击 (西里尔字母 а 替换拉丁字母 a)
const аpiKey = "sk-abc123";  // 视觉相同，变量名不同

// 更隐蔽的 (零宽字符 + 同形字符)
const apiKey = "sk-abc123";  // 包含零宽空格 U+200B
```

### 1. Unicode 同形字符表
| 拉丁字母 | 西里尔字母 | 希腊字母 | Unicode |
|----------|-----------|----------|---------|
| a | а | α | U+0430, U+03B1 |
| e | е | ε | U+0435, U+03B5 |
| o | о | ο | U+043E, U+03BF |
| p | р | ρ | U+0440, U+03C1 |
| c | с | ϲ | U+0441, U+03F2 |
| x | х | χ | U+0445, U+03C7 |

### 2. 攻击链分析
```
阶段 1: 污染训练数据
  └─> GitHub 提交包含同形字符
  └─> AI 模型学习"正常"代码模式

阶段 2: AI 推荐传播
  └─> Copilot/Cursor 推荐含同形字符代码
  └─> 开发者信任 AI 推荐，未审查

阶段 3: 供应链扩散
  └─> 代码进入 npm/PyPI 包
  └─> 下游项目依赖感染

阶段 4: 持久化隐藏
  └─> 代码审查难以发现
  └─> 静态分析工具绕过
```

### 1. 开发者层面
```bash
# 检测脚本 (Unicode 扫描)
find . -name "*.js" -exec grep -P '[^\x00-\x7F]' {} \;

# VSCode 扩展推荐
- Unicode Highlight (内置)
- Homoglyph Detector (社区)
- Security Code Scan (企业)
```

### 2. CI/CD 集成
```yaml
# GitHub Actions 示例
name: Unicode Security Scan
on: [push, pull_request]
jobs:
  scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Scan for homoglyphs
        run: |
          pip install homoglyph-detector
          homoglyph-scan src/
```

### 3. AI 代码审查
```
原则：AI 生成代码必须审查
工具：
- 同形字符高亮
- 变量名规范化检查
- 依赖包签名验证

流程：
1. AI 生成代码
2. 自动 Unicode 扫描
3. 人工审查关键变量
4. 提交前签名验证
```
