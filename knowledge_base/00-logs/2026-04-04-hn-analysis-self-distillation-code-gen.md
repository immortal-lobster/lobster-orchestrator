## 📌 核心发现

研究人员发现一种**极其简单**的自蒸馏方法（Simple Self-Distillation, SSD），无需验证器、教师模型或强化学习，仅用模型自己的输出进行微调，就能显著提升代码生成能力。

### 关键数据
| 模型 | 优化前 pass@1 | 优化后 pass@1 | 提升 |
|------|--------------|--------------|------|
| Qwen3-30B-Instruct | 42.4% | 55.3% | **+12.9%** |

增益集中在**更难的问题**上，且在 Qwen 和 Llama 的 4B/8B/30B 规模上都有泛化效果。

---

## 🔬 方法原理

### SSD 工作流程
```
1. 从模型采样解决方案（特定 temperature + truncation 配置）
2. 用这些样本进行标准监督微调（SFT）
3. 无需外部验证，无需奖励模型
```

### 为什么有效？

研究人员发现 LLM 解码中存在**精度 - 探索冲突**：
- **精度模式**: 需要抑制 distractor tails（干扰项）
- **探索模式**: 需要保留有用的多样性

SSD 通过**上下文相关的方式重塑 token 分布**：
- 在需要精度的地方，抑制干扰项
- 在需要探索的地方，保留多样性

### 1. 自进化的可能性

Sandbot 团队可以借鉴 SSD 思路：
- 记录 Agent 成功完成任务的轨迹

- 当前：轨迹记录 + 奖励反馈 + A/B 测试
- 可升级：加入 SSD 式自蒸馏微调

|------|------|------|----------|

SSD 提供了**性价比最高**的后训练优化方向。

## 📊 技术细节深入

### 采样配置建议

论文中提到"特定 temperature 和 truncation 配置"，虽然未详述，但基于类似研究：

```python
# 推荐的采样配置
temperature = 0.7-0.9  # 平衡探索与利用
top_p = 0.9-0.95       # 核采样，保留 90-95% 概率质量
max_tokens = 4096      # 足够长的代码生成
n_samples = 10-20      # 每个问题采样 10-20 个解
```

### 微调数据筛选

不是所有采样输出都适合微调：
- ✅ 保留：能通过单元测试/验证的解
- ✅ 保留：代码风格一致、可读性高的解
- ❌ 过滤：语法错误、逻辑错误的解
- ❌ 过滤：过度复杂、难以维护的解
