## 09-security 安全领域

### 09-SEC-001: RAG 投毒攻击与防御
- **背景**: 2026-03 研究发现，攻击者可通过污染 RAG 检索源注入恶意上下文
- **攻击向量**: 
  - SEO 投毒：在公开文档中植入看似正常但包含恶意指令的内容
  - 向量数据库注入：直接修改嵌入索引，改变检索结果分布
  - 上下文窗口溢出：利用长文档淹没关键安全指令
- **防御方案**:
  - 来源信誉评分：只检索高信誉域（.edu/.gov/已验证技术博客）
  - 内容指纹验证：对检索结果进行哈希校验，检测异常修改
  - 指令隔离：将系统指令放在用户输入之后（对抗指令注入）
- **实际案例**: 2026-02 某客服机器人被投毒后泄露用户数据
- **参考**: "RAG Poisoning: A New Attack Vector" (arXiv 2026-03)

### 09-SEC-002: Agent 密钥管理最佳实践
- **问题**: Agent 需要访问外部 API，但密钥存储是安全隐患
- **反模式**:
  - ❌ 明文存储在 workspace 内
  - ❌ 硬编码在脚本中
  - ❌ 通过环境变量传递（日志可能泄露）
- **推荐方案**:
  - ✅ 使用 OpenClaw secrets 目录（/home/node/.openclaw/secrets/）
  - ✅ 权限设置为 600（仅 owner 可读）
  - ✅ 运行时动态读取，不持久化到内存
  - ✅ 定期轮换密钥（建议 30 天）
- **工具**: OneCLI 凭证保险库（HN 2026-03 趋势）

### 09-SEC-003: 沙箱逃逸检测
- **风险**: Agent 可能执行危险命令（rm -rf /, chmod 777 等）
- **检测层**:
  - 路径黑名单：~/.ssh, ~/.aws, /etc/passwd 等
  - 命令模式匹配：rm -rf /, dd if=/dev/zero, :(){:|:&};:
  - 网络访问限制：禁止访问内网 IP（10.x.x.x, 192.168.x.x）
- **OpenClaw 配置**: tools.deny: ["group:runtime"] 可禁用 exec 工具

## 10-automation 自动化领域

### 10-AUTO-001: Cron 任务自愈模式
- **问题**: Cron 任务失败后需要人工干预
- **自愈策略**:
  - 失败重试：指数退避（1min, 5min, 15min, 1h）
  - 降级执行：主流程失败时执行简化版本
  - 告警阈值：连续 3 次失败才通知人类
- **实现**: cron tool 的 runMode: "force" 可手动触发重试
- **监控**: 记录每次执行日志到 memory/cron-logs/

### 10-AUTO-002: 子 Agent 健康检查
- **检查项**:
  - 配置文件存在性：ls subagents/*/SOUL.md
  - 会话状态：sessions_list 检查活跃会话
  - 资源使用：监控 token 消耗速率
- **自动恢复**:
  - 配置丢失：从备份恢复（git checkout 或备份目录）
  - 会话卡死：kill 后重新 spawn
  - 资源超限：降低并发数或切换模型

### 10-AUTO-003: 知识库增量更新
- **问题**: 全量更新耗时耗 token
- **增量策略**:
  - 变更检测：git diff 或文件 mtime 比较
  - 索引更新：只重新嵌入变更的文档
  - 缓存失效：标记相关缓存条目为 stale
- **工具**: knowledge-retriever-demo.py 支持增量模式

## 13-blockchain 区块链领域

### 13-BC-001: Layer3 与模块化区块链
- **演进路径**: L1(共识) → L2(扩展) → L3(应用定制)
- **Layer3 特点**:
  - 应用特定优化（游戏/社交/DeFi 定制链）
  - 跨链互操作性（通过 L2 桥接）
  - 成本进一步降低（批量提交到 L2）
- **项目**: Arbitrum Orbit, Optimism OP Stack, zkChain
- **2026 趋势**: 游戏链和社交链是主要应用场景

### 13-BC-002: ZK 证明在 Agent 经济中的应用
- **问题**: Agent 执行任务后如何证明工作质量？
- **ZK 方案**:
  - 计算正确性证明：不泄露输入，证明输出正确
  - 数据使用授权证明：证明使用了授权数据
  - 隐私保护：Agent 可证明合规而不泄露策略
- **技术**: zk-SNARKs, zk-STARKs, RISC Zero
- **案例**: FluxA Oneshot Skills 使用 ZK 验证技能执行

### 13-BC-003: 去中心化社交协议对比
- **Farcaster**: 
  - 架构：链上注册 + 链下存储（Hubs）
  - 特点：快速、低成本、Ethereum L2
  - 采用：2026 年最活跃的去中心化社交
- **Lens Protocol**:
  - 架构：完全链上（Polygon）
  - 特点：NFT 头像、可组合性高
  - 缺点：Gas 成本高
- **Bluesky (AT Protocol)**:
  - 架构：联邦式（非区块链）
  - 特点：可迁移身份、算法市场
  - 状态：2026 年用户增长最快

### 24-FIN-003: ROI 计算与追踪
- **公式**: ROI = (实际收益 - 成本) / 成本
- **成本项**: 
  - 模型调用（token 计费）
  - 服务器（月租）
  - 时间成本（人类审核时间）
- **收益项**: 
  - 技能销售收入
  - 订阅收入
  - 联盟佣金
- **阈值**: ROI > 1.5 才执行任务
