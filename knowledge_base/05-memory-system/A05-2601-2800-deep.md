### A05-2601: 记忆系统架构设计

**定义**: 记忆系统是 AI Agent 的长期知识存储与检索机制

**核心原理**: 
- 分层存储：核心记忆 (MEMORY.md) + 每日记忆 (memory/YYYY-MM-DD.md) + 知识库 (knowledge_base/)
- 写入策略：即时文档化 (Text > Brain)
- 检索策略：语义搜索 + 关键词匹配

**实际案例**: Sandbot V6.4.0 采用三层记忆架构，核心记忆 300 行以内，每日记忆自动归档，知识库 4760+ 文件/1M+ 知识点

**关键数据**: 
- 记忆检索延迟：<100ms (本地 grep)
- 记忆压缩率：10:1 (每日→核心提炼)
- 记忆保留周期：核心永久，每日 30 天归档

**应用场景**: AI Agent 身份保持、跨会话连续性、知识积累

**配置参数**: 记忆分层阈值、归档周期、压缩策略

**扩展阅读**: AGENTS.md (记忆闭环机制), SOUL.md (血泪教训)

### A05-2602: 记忆提炼与压缩

**定义**: 从原始日志中提炼核心教训存入长期记忆

**核心原理**:
- 触发条件：重大决策、核心教训、身份认知变更
- 提炼流程：读取每日记忆→识别模式→抽象原则→写入 MEMORY.md
- 压缩策略：删除过时信息、合并相似教训、保留可复用知识

**实际案例**: 2026-02-24 "18 天幻觉循环"教训从 2000 字对话提炼为 3 行核心原则刻入 SOUL.md

**关键数据**:
- 提炼频率：每 3-7 天执行一次
- 压缩比：20:1 (原始对话→核心教训)
- 核心记忆容量：300 行上限

**应用场景**: 避免重复犯错、知识沉淀、Agent 进化

**配置参数**: 提炼阈值、保留策略、版本标记

**扩展阅读**: memory/2026-02-24.md (原始对话), MEMORY.md (提炼结果)
