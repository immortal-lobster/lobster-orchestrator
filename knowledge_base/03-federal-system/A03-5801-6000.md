## 1. 多 Agent 共识的实际代价

**问题**: 7 个子 Agent 同时写入知识库，谁的结论优先？

**经典方案对比**:

| 方案 | 延迟 | 容错 | 适合场景 |
|------|------|------|----------|
| Leader-based (Raft) | 低 (单写入点) | Leader 挂了要选举 | 强一致性需求 |
| Leaderless (CRDT) | 极低 (本地写) | 任意节点可写 | 最终一致性即可 |
| 投票 (BFT) | 高 (2/3 共识) | 容忍 1/3 拜占庭 | 不信任环境 |

**Sandbot 实战选择**: 文件系统即共识层。每个 Cron 任务写独立文件 (无锁竞争)，审计 Agent 后置校验。这本质上是一种乐观并发 + 后验审计模式。

**Bram Cohen (Manyana) 的启发**: CRDT 合并永不失败，冲突变信息展示而非错误。对多 Agent 知识库写入的意义——两个 Agent 对同一话题写了不同结论，不是冲突，是多视角。

## 2. 联邦系统的真实故障模式

**Sandbot 遇到过的故障** (从 memory/ 日志提炼):

| 故障 | 原因 | 恢复方式 | 恢复时间 |
|------|------|----------|----------|
| 子 Agent 超时 | 上下文溢出 | 自动重试 + 截断输入 | 30s |
| 知识库文件冲突 | 两个 Cron 写同名文件 | 文件名加时间戳后缀 | 即时 |
| 记忆文件损坏 | write 工具中途中断 | 从前一版本恢复 | 5min |
| 外部 API 不可达 | DNS/服务下线 | 降级为本地知识 | 0 (优雅降级) |

**关键教训**: 联邦系统的可靠性不取决于单点不出错，而取决于出错后恢复多快。Sandbot 的 "先验证再汇报" 原则就是从 InStreet DNS 失败事件中学来的。

## 3. 状态同步：从 Supermemory ASMR 学到的

**传统方式**: 所有 Agent 共享一个状态存储 (如 Redis/数据库)  
**ASMR 方式**: 每个 Agent 独立观察 → 并行提取 → 专家聚合投票

**Supermemory 三层架构映射到联邦系统**:

```
层级 1: Observer Agents (并行摄取)
  → Sandbot: 每个 Cron 独立扫描 HN/Reddit/知识库
  → 不互相等待，不共享中间状态

层级 2: Search Agents (主动检索)  
  → Sandbot: grep + memory_search + 多角度查询
  → 不用向量数据库，Agent 推理即检索

层级 3: Answering Ensembles (集成回答)
  → Sandbot: 主 Agent 汇总子 Agent 产出
  → 投票/聚合/去重
```

**核心洞察**: 联邦系统的状态同步不需要实时一致，需要的是"最终可审计"。每个 Agent 写自己的 memory/YYYY-MM-DD.md，主 Agent 定期汇总，审计 Agent 后置校验。

## 4. 联邦扩展的天花板在哪里

**理论上**: N 个子 Agent 应该带来 N 倍产出  
**实际上**: 协调成本随 N² 增长

**Sandbot 7 Agent 联邦的实际瓶颈**:
1. **上下文竞争**: 每个子 Agent 需要读取公共文件 (SOUL.md, MEMORY.md)，形成读热点
2. **审计瓶颈**: 1 个 Auditor 审查 6 个 Agent 的产出，成为单点
3. **命名空间冲突**: 知识库目录需要预先划分，否则文件覆盖

**实用上限**: 对于文件系统协作模式，5-10 个 Agent 是甜点区。超过 10 个需要引入分片 (知识库按领域分区) 或层级联邦 (Agent 组长 + 组员)。

## 5. 设计联邦系统的 5 条铁律

从 Sandbot V6.0-V6.3 一年多实践中提炼:

1. **文件即接口**: Agent 之间通过文件通信，不通过内存共享。文件可审计、可回溯、可离线读取。
2. **乐观写入 + 后置审计**: 不在写入时加锁，在审计时发现问题。写入速度 > 一致性保证。
3. **独立故障域**: 每个子 Agent 崩溃不影响其他 Agent。Cron 任务之间无依赖。
4. **状态可重建**: 任何 Agent 丢失状态，可从 memory/ + knowledge_base/ 完全恢复。不依赖内存态。
5. **人类兜底**: 联邦系统的最终审批者是老大。自动化做 95%，人类决策 5% 关键点。
