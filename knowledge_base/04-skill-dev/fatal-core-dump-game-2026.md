## 🎮 游戏机制

### 核心玩法
```
1. 场景加载
   - 每个关卡是一个 C 程序崩溃场景
   - 提供源代码 (有 bug)
   - 提供 core dump 文件

2. 调试工具
   - gdb break <function>: 设置断点
   - gdb run: 运行程序
   - gdb bt: 查看调用栈
   - gdb print <var>: 查看变量
   - gdb info registers: 查看寄存器

3. 线索收集
   - 崩溃位置 (哪一行代码)
   - 错误变量 (哪个值不对)
   - 内存状态 (是否越界/泄漏)
   - 调用链 (函数调用顺序)

4. 推理结论
   - 选择 bug 类型 (空指针/越界/内存泄漏)
   - 定位具体代码行
   - 提交答案

5. 反馈学习
   - 正确答案：详细解释 + 知识点
   - 错误答案：提示 + 重试机会
```

### 关卡设计
```
Level 1: 空指针 dereference
  - 场景：程序访问 NULL 指针
  - 线索：Segmentation fault
  - 学习：gdb print 查看指针值

Level 2: 数组越界
  - 场景：访问数组边界外内存
  - 线索：随机值/崩溃
  - 学习：gdb print array[0]...array[n]

Level 3: 栈溢出
  - 场景：无限递归
  - 线索：Stack overflow
  - 学习：gdb bt 查看调用栈深度

Level 4: 内存泄漏
  - 场景：malloc 无 free
  - 线索：内存持续增长
  - 学习：valgrind 集成

Level 5: 竞态条件
  - 场景：多线程数据竞争
  - 线索：间歇性崩溃
  - 学习：gdb thread 查看线程状态

Level 6: 未定义行为
  - 场景：未初始化变量
  - 线索：随机行为
  - 学习：gdb watch 监视变量

Boss Level: 综合调试
  - 场景：多个 bug 组合
  - 线索：复杂崩溃现场
  - 学习：综合调试策略
```

## 🛠️ 技术实现

### 前端架构
```
界面布局:
  ┌─────────────────────────────────────┐
  │ 故事叙述 / 任务目标                  │
  ├─────────────────────────────────────┤
  │ 代码编辑器 (只读，语法高亮)          │
  ├─────────────────────────────────────┤
  │ GDB 命令行模拟器                     │
  │ $ gdb ./program                     │
  │ (gdb) break main                    │
  │ (gdb) run                           │
  ├─────────────────────────────────────┤
  │ 输出窗口 (程序输出 + GDB 响应)        │
  ├─────────────────────────────────────┤
  │ 变量监视器 / 内存查看器              │
  └─────────────────────────────────────┘
```

### GDB 模拟器
```
实现方式:
  - 不是真实 GDB (浏览器无法运行)
  - 预定义场景 + 脚本响应
  - 模拟 GDB 输出格式

命令支持:
  - break/b: 断点设置 (预定义位置)
  - run/r: 程序运行 (触发场景脚本)
  - continue/c: 继续执行
  - next/n: 单步执行
  - step/s: 步入函数
  - print/p: 变量查看 (预定义值)
  - backtrace/bt: 调用栈 (预定义)
  - info: 寄存器/内存信息
  - quit/q: 退出

响应逻辑:
  - 解析用户命令
  - 匹配预定义场景
  - 返回模拟 GDB 输出
  - 更新界面状态
```

### 游戏状态管理
```
状态机:
  - LOADING: 加载关卡资源
  - PLAYING: 调试进行中
  - SOLVED: 找到 bug
  - FAILED: 调试失败
  - HINT: 查看提示

持久化:
  - localStorage 保存进度
  - 已解锁关卡
  - 成就/徽章
  - 最佳用时
```

## 📚 教育价值分析

### 传统 GDB 教学 vs 游戏化学习
| 维度 | 传统教学 | Fatal Core Dump |
|------|----------|-----------------|
| 动机 | 外部 (考试/工作) | 内部 (好奇心/成就感) |
| 反馈 | 延迟 (作业批改) | 即时 (命令响应) |
| 失败成本 | 高 (挫败感) | 低 (重试无成本) |
| 场景 | 抽象示例 | 叙事驱动 |
| 进度 | 统一节奏 | 个性化 |
| 留存率 | ~30% | 预计~70% |

### 认知科学原理
```
1. 情境学习 (Situated Learning)
   - 知识嵌入具体场景
   - "谋杀侦探"角色代入
   - 调试 = 破案

2. 渐进式挑战 (Flow Theory)
   - 难度匹配技能水平
   - 清晰目标 + 即时反馈
   - 心流体验

3. 错误友好 (Productive Failure)
   - 错误是学习机会
   - 详细解释为什么错
   - 鼓励重试

4. 间隔重复 (Spaced Repetition)
   - 关键概念多关卡出现
   - 不同场景强化记忆
   - 长期记忆形成
```
