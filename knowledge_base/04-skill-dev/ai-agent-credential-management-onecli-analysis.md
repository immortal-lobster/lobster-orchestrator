## 🎯 核心问题

### AI Agent 凭证管理痛点
```
问题描述:
  - AI Agent 需要访问外部 API (LLM/数据库/云服务)
  - API keys/tokens 需要安全存储
  - 传统方案：环境变量/配置文件 (不安全)
  - 风险：密钥泄露/硬编码/轮换困难

真实案例:
  - GitHub 扫描发现 1000+ 仓库泄露 API keys
  - 平均泄露发现时间：90 天
  - 泄露后损失：平均 $50,000+ (滥用计费)

核心需求:
  1. 密钥不落地 (Agent 不接触真实密钥)
  2. 透明注入 (请求时自动替换)
  3. 访问审计 (谁/何时/用了什么)
  4. 快速轮换 (泄露时立即失效)
```

## 🏗️ OneCLI 架构设计

### 系统组件
```
┌─────────────────────────────────────────────────────────┐
│                    AI Agent                             │
│  (使用 FAKE_KEY, 不知道真实密钥)                         │
└────────────────────┬────────────────────────────────────┘
                     │ HTTP 请求 (含 FAKE_KEY)
                     ▼
┌─────────────────────────────────────────────────────────┐
│              OneCLI Gateway (Rust)                       │
│  - 监听端口：10255                                       │
│  - 功能：拦截请求/匹配 host&path/解密注入                │
│  - 加密：AES-256-GCM                                     │
└────────────────────┬────────────────────────────────────┘
                     │ HTTP 请求 (含真实密钥)
                     ▼
┌─────────────────────────────────────────────────────────┐
│              外部 API (OpenAI/Anthropic/等)               │
│  (接收真实密钥，正常响应)                                │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│           OneCLI Dashboard (Next.js)                     │
│  - 监听端口：10254                                       │
│  - 功能：凭证管理/访问日志/权限配置                      │
└─────────────────────────────────────────────────────────┘
```

### 工作流程
```
Step 1: 凭证存入
  用户 → Dashboard → 加密存储 (AES-256-GCM) → PGlite/PostgreSQL

Step 2: Agent 配置
  Agent 配置使用 placeholder: OPENAI_API_KEY=FAKE_KEY_123

Step 3: 请求拦截
  Agent 发送请求 → OneCLI Gateway (端口 10255)

Step 4: 凭证匹配
  Gateway 检查 host + path → 匹配凭证规则 → 解密真实密钥

Step 5: 透明注入
  Gateway 替换 FAKE_KEY → 真实密钥 → 转发到外部 API

Step 6: 响应返回
  外部 API 响应 → Gateway → Agent (无密钥泄露)
```

## 🔐 安全技术细节

### 加密方案
```
算法：AES-256-GCM
  - 密钥长度：256 位
  - 模式：GCM ( authenticated encryption)
  - IV: 12 字节随机生成 (每凭证独立)
  - Tag: 16 字节认证标签

密钥派生:
  - 主密钥：用户密码 + PBKDF2
  - 迭代次数：100,000 次
  - 盐：16 字节随机

存储格式:
  {
    "ciphertext": "<base64>",
    "iv": "<base64>",
    "tag": "<base64>",
    "salt": "<base64>"
  }
```

### 凭证匹配规则
```
规则结构:
  {
    "name": "openai-main",
    "host": "api.openai.com",
    "pathPrefix": "/v1/",
    "headerName": "Authorization",
    "headerPattern": "Bearer {CREDENTIAL}",
    "credentialId": "cred_abc123"
  }

匹配逻辑:
  1. 检查请求 host 是否匹配
  2. 检查请求 path 是否匹配前缀
  3. 检查请求 header 是否符合 pattern
  4. 匹配成功 → 解密凭证 → 注入请求

示例:
  请求：GET https://api.openai.com/v1/chat/completions
        Authorization: Bearer FAKE_KEY_123
  
  匹配：host=api.openai.com ✓
        pathPrefix=/v1/ ✓
        headerPattern=Bearer {CREDENTIAL} ✓
  
  注入：Authorization: Bearer sk-real-key-xyz
```

## 📊 性能与安全权衡

### 性能开销
```
延迟分析:
  - 凭证匹配：<1ms (内存查找)
  - 解密操作：~2ms (AES-256-GCM)
  - 总开销：~3ms/请求
  - 对比：API 请求通常 100-1000ms

吞吐量:
  - 单实例：1000+ 请求/秒
  - 瓶颈：加密解密 (非网络)
  - 扩展：多实例负载均衡

资源占用:
  - 内存：~50MB (含 Dashboard)
  - CPU: <5% (空闲), ~20% (高负载)
  - 磁盘：凭证存储 (KB 级)
```

### 安全边界
```
信任边界:
  ✅ Gateway 与 Agent 同机部署 (可信环境)
  ✅ 凭证加密存储 (即使数据库泄露也无法解密)
  ✅ Agent 不接触真实密钥 (减少泄露面)
  ⚠️ Gateway 被攻破 = 所有凭证泄露 (单点风险)

缓解措施:
  - 最小权限：每个凭证限定 host+path
  - 访问审计：所有请求记录日志
  - 快速轮换：泄露时立即失效凭证
  - 网络隔离：Gateway 仅监听 localhost
```

## 🎯 对 Sandbot 知识库的启示

### 知识访问凭证系统
```
需求分析:
  - 知识库需要访问控制 (免费 vs 付费用户)
  - API 访问需要认证 (防止滥用)
  - 使用量需要追踪 (计费依据)

OneCLI 借鉴:
  1. 透明凭证注入
     - 用户 API key 不直接暴露
     - 请求时验证并注入权限

  2. 细粒度权限
     - 按知识领域授权 (01-ai-agent vs 24-finance)
     - 按操作类型授权 (读 vs 写 vs 导出)

  3. 访问审计
     - 谁/何时/访问了什么
     - 异常访问检测 (高频/批量)
```

### 实现方案建议
```
架构:
  ┌─────────────┐    ┌──────────────┐    ┌─────────────┐
  │   用户 App   │───▶│  KB Gateway  │───▶│  知识库 API  │
  │ (含 API Key) │    │ (认证/限流)   │    │  (2,390 文件) │
  └─────────────┘    └──────────────┘    └─────────────┘

功能:
  1. API Key 验证 (JWT/HMAC)
  2. 限流控制 (100 请求/分钟/用户)
  3. 权限检查 (付费领域访问)
  4. 使用审计 (日志记录)

技术栈:
  - Gateway: Rust/Go (高性能)
  - 存储：PostgreSQL (用户/权限)
  - 缓存：Redis (限流计数)
```
