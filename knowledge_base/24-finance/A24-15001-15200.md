## 去中心化金融 (DeFi) (A24-15001~A24-15050)

### A24-15001: 自动做市商 (AMM) 机制设计
- **定义**: 使用数学公式自动定价的流动性池模型
- **核心**: 恒定乘积 (x*y=k)、集中流动性、动态费率
- **应用**: Uniswap V3、Curve、Balancer 等 DEX
- **参数**: 资本效率、无常损失、滑点控制

### A24-15002: 去中心化借贷协议
- **定义**: 无需中介的加密资产抵押借贷平台
- **核心**: 超额抵押、清算机制、利率模型 (利用度曲线)
- **应用**: Aave、Compound、MakerDAO
- **参数**: 抵押率 (>150%)、清算阈值、利率弹性

### A24-15003: 流动性挖矿与收益优化
- **定义**: 通过提供流动性获取协议代币奖励
- **核心**: 奖励分配、代币排放曲线、收益复投
- **应用**: Yield Farming、收益聚合器 (Yearn)
- **参数**: APY、代币解锁周期、智能合约风险

### A24-15004: 跨链桥安全架构
- **定义**: 实现不同区块链之间资产转移的协议
- **核心**: 锁定 - 铸造、轻客户端验证、多签验证
- **应用**: Wormhole、LayerZero、Hop Protocol
- **参数**: 安全性 (验证者数量)、延迟、费用

### A24-15005: MEV (最大可提取价值) 防护
- **定义**: 防止矿工/验证者通过交易排序提取价值
- **核心**: 私有交易池、公平排序、Flashbots
- **应用**: 三明治攻击防护、清算优化
- **参数**: 提取量、防护成本、延迟影响

## 央行数字货币 (CBDC) (A24-15051~A24-15100)

### A24-15051: 零售型 CBDC 架构设计
- **定义**: 面向公众发行的数字形式法定货币
- **核心**: 账户型 vs 代币型、双层运营、离线支付
- **应用**: 数字人民币 (e-CNY)、数字欧元
- **参数**: 并发 TPS (>10 万)、隐私保护、可编程性

### A24-15052: 批发型 CBDC 应用场景
- **定义**: 面向金融机构的大额数字央行货币
- **核心**: 实时全额结算 (RTGS)、跨境支付、证券结算
- **应用**: 央行间结算、银行间市场
- **参数**: 结算最终性、互操作性、合规性

### A24-15053: CBDC 隐私与反洗钱平衡
- **定义**: 在保护用户隐私的同时满足监管要求
- **核心**: 可控匿名、分级 KYC、交易追踪阈值
- **应用**: 小额匿名、大额可追溯
- **参数**: 匿名限额、数据保留期、执法接口

### A24-15054: CBDC 跨境支付互操作
- **定义**: 不同国家 CBDC 系统之间的互联互通
- **核心**: 多边央行数字货币桥 (mBridge)、API 标准
- **应用**: 跨境贸易结算、汇款
- **参数**: 结算时间 (<10 秒)、成本、外汇兑换

### A24-15055: CBDC 对商业银行影响
- **定义**: CBDC 发行对传统银行体系的冲击与应对
- **核心**: 存款分流、准备金管理、信贷收缩风险
- **应用**: 持有限额、分级利率、银行中介角色
- **参数**: 持有上限、转换成本、货币政策传导

## AI 驱动的金融服务 (A24-15101~A24-15150)

### A24-15101: 智能投顾算法
- **定义**: 基于现代投资组合理论的自动化资产配置
- **核心**: 风险测评、均值 - 方差优化、再平衡策略
- **应用**: Betterment、Wealthfront、机器人理财
- **参数**: 夏普比率、最大回撤、管理费 (<0.5%)

### A24-15102: 信贷风控机器学习模型
- **定义**: 使用 ML 评估借款人违约风险
- **核心**: 特征工程、XGBoost/LightGBM、可解释性 (SHAP)
- **应用**: 消费贷、小微企业贷、反欺诈
- **参数**: AUC (>0.75)、KS 值 (>0.3)、坏账率

### A24-15103: 高频交易策略优化
- **定义**: 毫秒级执行的量化交易策略
- **核心**: 市场微观结构、订单簿分析、低延迟架构
- **应用**: 做市、套利、趋势跟踪
- **参数**: 延迟 (<100μs)、胜率、盈亏比

### A24-15104: 智能合约形式化验证
- **定义**: 使用数学方法证明智能合约无漏洞
- **核心**: 模型检测、定理证明、符号执行
- **应用**: DeFi 协议审计、保险合约
- **参数**: 覆盖率、误报率、验证时间

### A24-15105: 监管科技 (RegTech) 自动化
- **定义**: 使用技术自动化合规报告与监控
- **核心**: 自然语言处理 (法规解析)、异常检测、报告生成
- **应用**: AML 监控、交易报告、资本充足率计算
- **参数**: 准确率、误报率、人工复核率

## 金融安全与风险管理 (A24-15151~A24-15200)

### A24-15151: 实时欺诈检测系统
- **定义**: 在交易完成前识别并阻止欺诈行为
- **核心**: 规则引擎、行为分析、图神经网络
- **应用**: 信用卡盗刷、账户接管、洗钱
- **参数**: 检测延迟 (<100ms)、召回率 (>95%)、误报率 (<1%)

### A24-15152: 压力测试与情景分析
- **定义**: 评估金融机构在极端市场条件下的韧性
- **核心**: 宏观经济情景、风险因子冲击、资本充足率
- **应用**: 巴塞尔 III 合规、内部风险管理
- **参数**: 置信水平 (99%)、时间 horizon(1 年)、损失分布

### A24-15153: 模型风险管理框架
- **定义**: 管理金融模型的开发、验证、监控全生命周期
- **核心**: 模型清单、独立验证、性能监控、文档化
- **应用**: 监管合规 (SR 11-7)、模型治理
- **参数**: 验证频率、性能阈值、升级流程

### A24-15154: 网络安全事件响应
- **定义**: 应对金融系统网络攻击的应急响应流程
- **核心**: 检测、遏制、根除、恢复、事后分析
- **应用**: DDoS 攻击、数据泄露、勒索软件
- **参数**: 响应时间 (<1 小时)、RTO(<4 小时)、RPO(<15 分钟)

### A24-15155: 第三方风险管理
- **定义**: 评估和管理金融服务供应商的风险
- **核心**: 尽职调查、持续监控、退出计划
- **应用**: 云服务、支付处理、数据供应商
- **参数**: 风险评估频率、关键性分级、替代方案
