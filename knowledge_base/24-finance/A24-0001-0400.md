## 1. 算法交易：从理论到实际盈亏

### 算法交易不等于印钞机

很多入门文档把算法交易描述为"用代码自动赚钱"，但现实是：

| 理论描述 | 实际情况 |
|----------|----------|
| 回测年化 30% | 实盘滑点 + 手续费吃掉一半 |
| 策略越复杂越好 | 过拟合是最大杀手 |
| 高频 = 高利润 | 延迟竞赛只有头部玩家能赢 |
| 历史数据可靠 | 幸存者偏差 + 制度变更 |

### 实际策略分类与适用场景

**趋势跟踪（CTA 类）**：
- 原理：价格突破移动均线时入场
- 适合：期货、外汇等趋势明显的市场
- 痛点：震荡行情反复止损（2024-2025 年 A 股 CTA 策略普遍回撤 15-30%）
- 实操关键：止损幅度 = ATR × 2，持仓周期 5-20 天

**统计套利（配对交易）**：
- 原理：找到价格协整的两个标的，偏离时做多/空
- 适合：同行业股票、ETF 与成分股
- 痛点：协整关系会断裂（2023 年中概股 ADR/H 股配对策略大面积失效）
- 实操关键：协整检验 p-value < 0.05，半衰期 < 20 天才值得做

**做市策略**：
- 原理：同时挂买卖单赚 bid-ask spread
- 适合：流动性差的标的（小市值加密货币、场外期权）
- 痛点：极端行情单边吃亏（2022 年 LUNA 崩盘，做市商损失数十亿）
- 实操关键：库存风险对冲 + 毫秒级撤单能力

### 给 AI Agent 开发者的启示

算法交易的教训对 Agent 工程有直接借鉴：
1. **回测 ≠ 实盘** → 基准测试 ≠ 生产环境
2. **过拟合是最大敌人** → Agent 在特定 prompt 上表现好不代表泛化能力强
3. **风控比策略重要** → Agent 的安全边界比功能扩展重要

## 2. DeFi：去中心化金融的真实状态

### DeFi 核心协议对比（2026 年数据）

| 协议类型 | 代表项目 | TVL 规模 | 年化收益 | 核心风险 |
|----------|----------|----------|----------|----------|
| 借贷 | Aave V4 | ~$15B | 2-8% | 清算级联 |
| DEX | Uniswap V4 | ~$8B | 5-30% (LP) | 无常损失 |
| 稳定币 | MakerDAO | ~$12B | 3-5% (DSR) | 脱锚风险 |
| 收益聚合 | Yearn V3 | ~$2B | 5-15% | 合约漏洞 |
| 衍生品 | dYdX V4 | ~$1B | - | 流动性风险 |

### 被高估和被低估的 DeFi 概念

**被高估**：
- **TVL 作为项目指标**：可以通过递归借贷人为放大（实际资金可能只有显示的 1/3）
- **年化收益**：高 APY 往往来自代币通胀激励，扣除代币贬值后实际收益可能为负
- **去中心化**：大多数 DeFi 协议仍有管理员密钥、可升级合约，实际是"半中心化"

**被低估**：
- **可组合性**：DeFi 的真正创新不是单个协议，而是协议间可以像乐高一样组合
- **透明性**：所有交易链上可查，比传统金融透明度高几个数量级
- **7×24 运行**：无停盘、无交割日限制，全球统一市场

### 实践建议

1. **评估 DeFi 项目时**：看实际收入（协议费用）而非 TVL 或代币市值
2. **参与 DeFi 时**：优先选择经过 2+ 年运行、有审计的蓝筹协议
3. **构建 DeFi Agent 时**：必须实现滑点保护 + Gas 预估 + 交易模拟（先 `eth_call` 再实际发送）

## 3. 风险管理：VaR 之外的真实世界

### VaR 的局限性（2008 年的教训仍然适用）

Value at Risk (VaR) 是最常用的风险指标，但有致命缺陷：

```
VaR 说的是："在 95% 置信度下，明天最多亏 X 元"
VaR 没说的是："那 5% 的情况下会亏多少？"

答案可能是：10X、100X、甚至爆仓
```

**CVaR（条件 VaR）** 弥补了这个缺陷——它计算的是"最坏 5% 情况下的平均损失"。

### 风险管理实操框架

| 层级 | 工具 | 频率 | 目标 |
|------|------|------|------|
| 头寸级 | 止损单 | 实时 | 限制单笔损失 |
| 组合级 | VaR/CVaR | 日频 | 控制整体风险 |
| 压力测试 | 情景分析 | 周频 | 极端行情准备 |
| 战略级 | 风险预算 | 月频 | 资产配置调整 |

### 给 Agent 系统的风险管理借鉴

金融风控思维可以直接用于 Agent 系统：
- **止损 = 超时机制**：单次任务执行超过阈值就中断
- **VaR = Token 预算**：设定每次对话的最大 token 消耗
- **压力测试 = 对抗测试**：用恶意 prompt 测试 Agent 边界
- **风险预算 = 资源分配**：给不同子 Agent 分配不同的工具权限和资源上限

## 4. 监管科技（RegTech）：2026 年关键变化

### SEC 季报改革对算法交易的影响

2026 年 SEC 推进的季报格式标准化（XBRL 增强），直接影响：
- **量化策略**：更结构化的财报数据 → 因子提取更高效
- **另类数据**：SEC 对另类数据使用的监管趋严 → 合规成本上升
- **AI 审计**：SEC 开始要求披露 AI 模型在投资决策中的使用情况

### 对 Agent 开发者的影响

如果你在构建金融类 Agent：
1. 确保数据来源合规（不使用非公开信息）
2. 保留完整决策日志（可审计）
3. 关注各国对 AI 金融应用的监管动态（欧盟 AI Act 对高风险金融 AI 的要求）
