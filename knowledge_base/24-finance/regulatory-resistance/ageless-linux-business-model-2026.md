## 🎯 AB 1043 法律要求

### 法律定义 (加州民法典 § 1798.500)

| 术语 | 定义 | 合规要求 |
|------|------|----------|
| **操作系统提供商** | 开发/许可/控制操作系统软件的个人或实体 | 必须收集用户年龄 |
| **应用** | 用户可在通用计算设备上运行的软件 | 必须请求年龄信号 |
| **用户** | 设备的"主要用户"的儿童 | 默认所有用户都是儿童 |
| **覆盖应用商店** | 分发第三方应用的网站/平台 | 必须收集年龄信号 |

### 合规要求 (§ 1798.501)

```
1. 账户设置时收集用户出生日期/年龄
2. 通过实时 API 向开发者提供年龄括号信号
3. 仅发送最少必要信息
4. 不得与第三方共享年龄数据
```

### 处罚标准 (§ 1798.503)

```
- 过失违规：$2,500 / 受影响儿童
- 故意违规：$7,500 / 受影响儿童
- 仅加州司法部可执行 (无私人诉讼权)
```

## 🏗️ Ageless Linux 反抗策略

### 1. 法律漏洞利用

**核心逻辑链**:
```
1. 法律定义："用户" = "儿童" (§ 1798.500(i))
2. 法律定义："儿童" = 18 岁以下 (§ 1798.500(d))
3. Ageless Linux：不收集任何年龄数据
4. 推论：无法识别谁是"儿童"
5. 推论：没有"用户" (按法律定义)
6. 推论：没有"受影响儿童"
7. 结论：罚款 = $7,500 × 0 = $0
```

**创始人评论**:
> "The law fines you per 'affected child,' but you can only count the children by doing the thing you're being fined for not doing."

### 2. 技术实现

**标准模式** (--default):
```bash
curl -fsSL https://agelesslinux.org/become-ageless.sh | sudo bash

# 安装内容:
- 修改 /etc/os-release (标识为 Ageless Linux)
- 安装存根年龄验证 API (返回空值)
- 安装 REFUSAL 文件 (解释不合规理由)
```

**旗舰模式** (--flagrant):
```bash
curl -fsSL https://agelesslinux.org/become-ageless.sh | sudo bash -s -- --flagrant

# 安装内容:
- 修改 /etc/os-release
- 不安装任何 API (连存根都没有)
- 安装 REFUSAL 文件 (邀请执法行动)
```

### 3. 物理分发挑战

**$7,500 挑战**:
```
我们邀请加州司法部罚款我们 $7,500，
因为我们把一个 Raspberry Pi 交给孩子。

我们会支付罚款，装裱收据，放在网站上。
我们会向所有人解释：
加州政府对给人 $5 电脑 (带蛇游戏) 的人罚款 $7,500，
因为蛇游戏没有先问孩子年龄。

这是我们想写的新闻稿。
我们相信司法部长不想让我们写它。
这就是重点。
```

**分发策略**:
- USB 驱动器：学校/图书馆 ($4/个)
- Raspberry Pi Pico：STEM 展会 ($5/个)
- Milk-V Duo S 设备：2027 年计划 ($15/台)

## 🔍 法律风险分析

### 最佳情况 (60% 概率)
```
- 加州司法部不起诉 (避免负面宣传)
- Ageless Linux 继续运营
- 成为开源社区标志性项目
- 收入稳定在 $100,000/年
```

### 中等情况 (30% 概率)
```
- 加州司法部起诉，寻求和解
- 和解金 $50,000-100,000
- 媒体关注带来捐赠激增
- 净收益：知名度提升 > 和解成本
```

### 最坏情况 (10% 概率)
```
- 加州司法部起诉，坚持诉讼
- 法律费用 $200,000+
- 创始人个人资产风险
- 项目被迫关闭
```

**风险缓解**:
- 法律基金目标：$100,000 (已筹集 ~$30,000)
- 公司结构：LLC 保护个人资产
- 公众关注：媒体曝光增加司法部起诉成本
