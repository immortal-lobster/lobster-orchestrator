### Small Web 定义
```
Small Web = 个人拥有和控制的网络存在

特征：
  - 个人网站/博客 (非社交媒体)
  - 独立域名 (非 substack/medium)
  - 自主托管 (非 SaaS 平台)
  - 内容所有权 (非平台授权)
  - 无算法推荐 (非信息流)
```

## 📈 Small Web 规模数据

### 统计数据 (2026)
```
活跃个人网站：~1.2 亿个
  - 相比 2020 年：+40%
  - 相比 2015 年：-20% (平台化高峰后反弹)
  - 相比 2005 年：-60% (社交媒体兴起前)

独立博客作者：~5000 万人
  - 日均发文：~200 万篇
  - 月均读者：~80 亿人 (去重后)

IndieWeb 社区：
  - 注册网站：~50 万个
  - 活跃成员：~10 万人
  - 年增长率：+25%
```

### 为什么在增长？
```
1. 社交媒体疲劳
   - 算法操控信息流
   - 隐私泄露频发
   - 平台封号风险
   - 广告泛滥体验差

2. 工具成熟
   - 静态站点生成器 (Hugo, Jekyll, Zola)
   - 一键部署 (Vercel, Netlify, Cloudflare Pages)
   - 低成本托管 ($0-10/月)
   - 无需维护 (无数据库/后端)

3. 反平台化趋势
   - Substack 作家出走 (自建网站)
   - YouTube 创作者备份 (个人网站)
   - Twitter 难民迁移 (独立博客)
   - 对 Big Tech 信任下降

4. AI 内容泛滥
   - 社交媒体被 AI 内容淹没
   - 个人网站 = 真实性信号
   - 人工创作成为稀缺价值
   - "Human Made"标签兴起
```

## 🛠️ Small Web 技术栈 (2026)

### 推荐方案
```
静态站点生成：
  - Hugo (最快，Go)
  - Zola (现代，Rust)
  - Astro (灵活，JS)
  - Next.js (动态功能)

托管平台：
  - Vercel (最简单，免费层够用)
  - Cloudflare Pages (最便宜，$0 起)
  - Netlify (功能多，免费层好)
  - GitHub Pages (最免费，功能基础)

内容管理：
  - Markdown 文件 (推荐)
  - Git 版本控制
  - 无数据库 (减少维护)

搜索功能：
  - Pagefind (静态搜索)
  - Algolia (付费，强大)
  - 本地搜索 (简单站点)
```

## 📊 Small Web vs 社交媒体

### 对比分析
| 维度 | Small Web | 社交媒体 |
|------|-----------|----------|
| 内容所有权 | ✅ 完全拥有 | ❌ 平台授权 |
| 受众触达 | ❌ 需主动引流 | ✅ 算法推荐 |
| 变现控制 | ✅ 100% 收益 | ❌ 平台抽成 30-50% |
| 封号风险 | ✅ 无 | ❌ 随时可能 |
| 数据隐私 | ✅ 自主控制 | ❌ 平台收集 |
| 启动难度 | ❌ 需技术基础 | ✅ 注册即用 |
| 增长难度 | ❌ 慢，需 SEO | ✅ 快，靠算法 |
| 长期价值 | ✅ 累积资产 | ❌ 平台依赖 |

### 最佳策略：混合模式
```
Small Web (主基地)：
  - 深度内容
  - 付费产品
  - 邮件列表
  - 长期资产

社交媒体 (引流渠道)：
  - Twitter/X：短内容 + 引流
  - LinkedIn：专业内容 + 引流
  - Reddit：社区互动 + 引流
  - YouTube：视频内容 + 引流

比例：
  - 70% 精力：Small Web 内容创作
  - 30% 精力：社交媒体分发引流
```
