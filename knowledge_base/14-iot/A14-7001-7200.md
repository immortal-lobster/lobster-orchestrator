## A14-7001 到 A14-7050: IoT 通信协议

### A14-7001: MQTT (Message Queuing Telemetry Transport)
- **定义**: 轻量级发布/订阅消息协议
- **端口**: 1883 (明文), 8883 (TLS)
- **QoS 级别**: 0 (至多一次), 1 (至少一次), 2 (恰好一次)
- **应用**: 传感器数据上报、远程控制、边缘计算
- **优势**: 低带宽、低功耗、适合不稳定网络

### A14-7002: CoAP (Constrained Application Protocol)
- **定义**: 专为受限设备设计的 RESTful 协议
- **传输**: UDP (默认), TCP/TLS/WebSocket
- **方法**: GET, POST, PUT, DELETE
- **应用**: 智能家居、工业传感器、低功耗设备
- **对比 HTTP**: 头部开销小 (4 字节 vs 数百字节)

### A14-7003: LoRaWAN (Long Range Wide Area Network)
- **定义**: 低功耗广域网协议
- **频段**: 868MHz (EU), 915MHz (US), 470MHz (CN)
- **覆盖**: 城市 2-5km, 郊区 10-15km
- **功耗**: 电池寿命 5-10 年
- **应用**: 智能电表、农业监测、资产追踪

### A14-7004: NB-IoT (Narrowband IoT)
- **定义**: 蜂窝网络低功耗广域技术
- **带宽**: 180kHz (窄带)
- **优势**: 运营商网络、室内覆盖好、移动性强
- **应用**: 智能停车、消防监测、共享单车
- **对比 LoRa**: 需要 SIM 卡、有资费、但覆盖更广

### A14-7005: Zigbee (IEEE 802.15.4)
- **定义**: 短距离低功耗 mesh 网络协议
- **频段**: 2.4GHz (全球), 868MHz (EU), 915MHz (US)
- **拓扑**: 星型、树型、Mesh
- **节点数**: 最多 65000 个
- **应用**: 智能家居 (Philips Hue, IKEA Tradfri)

## A14-7051 到 A14-7100: IoT 硬件与传感器

### A14-7051: ESP32 微控制器
- **核心**: 双核 Xtensa LX6, 240MHz
- **连接**: WiFi 802.11 b/g/n, Bluetooth 4.2/BLE
- **内存**: 520KB SRAM, 4MB Flash (典型)
- **功耗**: 深度睡眠 10μA
- **应用**: 智能家居节点、传感器网关

### A14-7052: Raspberry Pi (边缘网关)
- **型号**: Pi 4B (4GB), Pi Zero 2 W, Pi 5
- **系统**: Raspberry Pi OS (Debian)
- **接口**: GPIO, I2C, SPI, UART, USB
- **应用**: 边缘计算、本地 MQTT Broker、Home Assistant

### A14-7053: 温湿度传感器 (DHT22/AM2302)
- **范围**: 温度 -40~80°C, 湿度 0-100%RH
- **精度**: ±0.5°C, ±2%RH
- **接口**: 单总线
- **采样率**: 0.5Hz (每 2 秒)
- **应用**: 环境监测、农业温室

### A14-7054: 加速度计/陀螺仪 (MPU6050)
- **传感器**: 3 轴加速度 +3 轴陀螺仪
- **量程**: ±2/4/8/16g, ±250/500/1000/2000°/s
- **接口**: I2C (0x68/0x69)
- **应用**: 姿态检测、振动监测、计步器

### A14-7055: GPS 模块 (NEO-6M/NEO-M8N)
- **系统**: GPS, GLONASS, Galileo, BeiDou
- **精度**: 2.5m CEP (NEO-M8N)
- **更新率**: 1-10Hz
- **接口**: UART (NMEA 0183)
- **应用**: 车辆追踪、无人机导航、资产定位

## A14-7101 到 A14-7150: IoT 平台与云服务

### A14-7101: AWS IoT Core
- **功能**: 设备连接、消息路由、设备管理
- **协议**: MQTT, HTTPS, WebSockets
- **安全**: X.509 证书、IAM 策略
- **集成**: Lambda, DynamoDB, Kinesis
- **定价**: $1/百万消息 (典型)

### A14-7102: Azure IoT Hub
- **功能**: 双向通信、设备管理、监控
- **层级**: Free (8KB/天), B1, S1-3
- **特性**: 设备 twins, 直接方法，云到设备消息
- **集成**: Azure Functions, Stream Analytics
- **优势**: 企业级、与 Azure 生态深度集成

### A14-7103: Google Cloud IoT Core (已退役，迁移建议)
- **状态**: 2023 年 8 月退役
- **替代**: Cloud Pub/Sub + Device SDK
- **迁移路径**: 设备认证→Cloud IoT, 数据→Pub/Sub
- **教训**: 避免单点依赖、设计可迁移架构

### A14-7104: ThingsBoard (开源平台)
- **功能**: 设备管理、数据可视化、规则引擎
- **部署**: Docker, Kubernetes, 云托管
- **协议**: MQTT, CoAP, HTTP
- **优势**: 自托管、数据主权、成本低
- **应用**: 中小企业、私有云部署

### A14-7105: Home Assistant (智能家居)
- **定位**: 本地优先的智能家居平台
- **集成**: 1000+ 设备/服务
- **自动化**: YAML 配置、可视化编辑器
- **优势**: 隐私保护、离线可用、无订阅费
- **硬件**: Home Assistant Green, Yellow, DIY

## A14-7151 到 A14-7200: IoT 安全与最佳实践

### A14-7151: 设备身份认证
- **方法**: X.509 证书、预共享密钥、TPM
- **最佳实践**: 每设备唯一凭证、定期轮换
- **风险**: 硬编码密钥、默认密码
- **案例**: Mirai 僵尸网络 (默认密码攻击)

### A14-7152: 安全启动 (Secure Boot)
- **定义**: 验证固件签名后才执行
- **实现**: Boot ROM→BL1→BL2→OS
- **硬件**: ARM TrustZone, Intel Boot Guard
- **价值**: 防止固件篡改、rootkit

### A14-7153: OTA 固件更新
- **要求**: 签名验证、回滚保护、断点续传
- **协议**: HTTPS, MQTT (大文件分块)
- **最佳实践**: A/B 分区、灰度发布
- **案例**: Tesla OTA、智能家居设备更新

### A14-7154: 数据加密
- **传输**: TLS 1.2/1.3, DTLS (UDP)
- **存储**: AES-256, 密钥管理系统
- **端到端**: 设备→云全程加密
- **合规**: GDPR, HIPAA, 等保 2.0

### A14-7155: IoT 安全框架 (NIST IR 8259)
- **核心能力**: 设备识别、漏洞管理、数据安全
- **基线**: 无硬编码密码、可更新、最小权限
- **评估**: CVSS 评分、渗透测试
- **认证**: ioXt, UL CAP, ETSI EN 303 645
