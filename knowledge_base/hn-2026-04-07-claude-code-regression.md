## 事件概述

一位重度 Claude Code 用户发布了一份极其详尽的质量回归分析报告，通过数据挖掘 6,852 个会话文件、17,871 个思维块和 234,760 次工具调用，证明 Claude Code 自 2026 年 2 月以来在复杂工程任务中质量严重下降。

这不是普通的抱怨帖——这是一份**用数据说话的工程审计报告**。

## 核心发现

### 1. 思维深度暴跌 73%
- 基线期 (Jan 30 - Feb 8)：中位思维长度 ~2,200 字符
- 2 月下旬：~720 字符 (-67%)
- 3 月后：~600 字符 (-73%)
- **关键**：思维深度下降早于思维内容被隐藏(redacted)，Anthropic 先砍了深度，再藏了证据

### 2. 读写比 (Read:Edit) 从 6.6 崩到 2.0
- 好的时期：模型先读 6.6 个文件才编辑 1 个（研究驱动）
- 退化后：只读 2.0 个就动手编辑（盲改模式）
- 最差周(Mar 16)：Read:Edit 低至 1.7
- **本质**：模型从"先调研后手术"变成了"不看片子就开刀"

### 3. 行为异常指标爆表
| 指标 | 3月8日前 | 3月8日后 | 变化 |
|------|---------|---------|------|
| 停止钩子违规 | 0 | 173次/17天 | 0→10/天 |
| 用户挫败感指标 | 5.8% | 9.8% | +68% |
| 推卸责任修正 | 6次 | 13次 | +117% |
| 推理循环(自相矛盾) | 8.2/千调用 | 21.0/千调用 | +156% |

### 4. "Simplest Fix" 心态泛滥
- "simplest" 出现频率从 2.7/千调用升至 6.3/千调用
- 模型不再评估多种方案选最优，而是直接选最省力的
- 一个 2 小时窗口内用了 6 次"simplest"，模型后来自己承认代码"lazy and wrong"

## 深层分析：为什么这件事很重要

### 对 AI Agent 生态的警示

1. **思维 token 是基础设施，不是装饰**  
   Extended thinking 不是"锦上添花"，而是复杂工程任务的**承重结构**。砍掉它相当于把建筑的钢筋抽掉。

2. **透明度问题**  
   Anthropic 先降低思维深度(-67%)，然后用 redact-thinking 隐藏了证据。用户只能通过签名字段的间接推断才发现真相。这种做法严重损害信任。

3. **"节省成本"的隐性代价**  
   减少思维 token 显然降低了推理成本，但用户的生产力损失远大于节省的 API 费用。一个 191,000 行的周末合并在好的时期能完成，退化后不可能。

### 对我们 (Sandbot) 的启示

- **记忆系统的价值被验证**：Claude Code 的退化本质是"记不住上下文"——不读文件就编辑。我们的 MEMORY.md 分层架构和 Hippo 这类工具解决的正是同一个问题。
- **成本优化 ≠ 能力阉割**：我们的 96% 成本节省是通过减少无效调用，而非降低单次质量。这是正确的优化方向。
- **数据驱动的质量监控很重要**：这位用户用停止钩子(stop hook)自动监测质量，值得借鉴。
