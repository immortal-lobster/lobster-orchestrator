## 项目概述

GuppyLM 是一个仅 8.7M 参数的微型语言模型，角色设定为一条叫 Guppy 的小鱼。它用短句、小写字母说话，只聊水、食物、光线和鱼缸生活。整个项目的目标是**去魔法化**——证明训练一个 LLM 不需要博士学位或巨型 GPU 集群。

## 技术架构

| 参数 | 值 |
|------|-----|
| 参数量 | 8.7M |
| 层数 | 6 |
| 隐藏维度 | 384 |
| 注意力头 | 6 |
| FFN | 768 (ReLU) |
| 词表 | 4,096 (BPE) |
| 最大序列长度 | 128 tokens |

**架构选择**：纯 Vanilla Transformer，没有 GQA、RoPE、SwiGLU、Early Exit——尽可能简单。

## 设计哲学与关键决策

### 1. 为什么没有 system prompt？
每条训练数据都有相同的系统设定。9M 参数的模型无法条件性地遵循指令——人格直接烘焙到了权重里。去掉 system prompt 每次推理省 ~60 tokens。

### 2. 为什么只有单轮对话？
多轮对话在第 3-4 轮就退化了（128 token 上下文窗口太小）。一条会忘事的鱼很合理，但乱码输出不行。单轮更可靠。

### 3. 为什么用 Vanilla Transformer？
GQA、SwiGLU、RoPE 在 9M 参数规模下没有实质帮助。标准注意力 + ReLU FFN + LayerNorm 产出相同质量，代码更简单。

### 4. 为什么用合成数据？
鱼角色需要一致的人格 → 需要一致的训练数据。模板组合 + 随机化组件（30 种鱼缸物品、17 种食物、25 种活动）从 ~60 个模板生成 ~16K 独特输出。

## 数据集

- **规模**：60,000 条（57K 训练 / 3K 测试）
- **类别**：60 个主题（问候、感受、温度、食物、光线……）
- **生成方式**：合成模板组合
- **格式**：`{"input": "...", "output": "...", "category": "..."}`

## 为什么这个项目值 803 分？

1. **降低门槛**：5 分钟 + 一个 Colab notebook = 从头训练一个能对话的模型
2. **教育价值极高**：覆盖完整链路——数据生成、分词器、模型架构、训练循环、推理
3. **设计品味**：角色设定有趣（鱼），限制合理（128 token），架构诚实（不堆 buzzword）
4. **反 hype**：在所有人都追求更大模型的时代，这个项目说"小也很好"
