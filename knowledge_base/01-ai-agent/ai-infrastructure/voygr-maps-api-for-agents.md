## 🗺️ 产品定位

### 问题陈述
```
传统地图 API (Google Maps, Mapbox) 的问题:

1. 为人类 UI 优化
   - 返回大量视觉信息 (图标、样式)
   - AI 用不上，浪费 token 和带宽

2. 查询语言不友好
   - REST API 复杂
   - 需要多次调用获取完整信息
   - 错误处理不清晰

3. 成本结构不合理
   - 按调用次数计费
   - AI Agent 高频调用成本爆炸
   - 没有批量优惠

4. 数据格式不标准
   - JSON 结构复杂嵌套
   - 字段命名不一致
   - 缺少语义标注
```

### Voygr 解决方案
```
核心特性:

1. AI-Native 数据格式
   - 精简 JSON (只返回 AI 需要的字段)
   - 语义标注 (机器可读的元数据)
   - 标准化 schema (跨平台兼容)

2. Agent 优化查询
   - 自然语言查询支持
   - 批量查询 (一次获取多类型数据)
   - 增量更新 (只返回变化部分)

3. 合理定价
   - 按数据量计费 (非调用次数)
   - AI Agent 专属套餐
   - 免费额度充足 (10K 调用/月)

4. 开发者体验
   - 完善的 SDK (Python/JS/Go)
   - 详细的文档和示例
   - 活跃的社区支持
```

## 📊 技术架构

### 数据层
```
数据源:
  - OpenStreetMap (基础地理)
  - 卫星图像 (遥感数据)
  - 政府公开数据 (道路/建筑)
  - 众包更新 (实时变化)

数据处理:
  - 自动化 ETL 管道
  - 质量验证 (多源交叉)
  - 实时更新 (分钟级延迟)
  - 版本控制 (历史回溯)
```

### API 层
```
端点设计:

GET /v1/location
  - 输入：坐标/地址/POI 名称
  - 输出：标准化位置信息
  - 延迟：<100ms

GET /v1/route
  - 输入：起点、终点、偏好
  - 输出：优化路径 (多模式)
  - 延迟：<200ms

GET /v1/search
  - 输入：自然语言查询
  - 输出：相关地点列表
  - 延迟：<150ms

POST /v1/batch
  - 输入：批量查询数组
  - 输出：批量结果
  - 延迟：<500ms (100 查询)
```

### SDK 示例
```python
# Voygr Python SDK
from voygr import VoygrClient

client = VoygrClient(api_key="your_key")

# 自然语言查询
result = client.search("coffee shops near Stanford University")

# 批量查询
locations = client.batch([
    {"query": "restaurants in Palo Alto"},
    {"query": "gas stations on route to SF"},
    {"query": "hotels near SFO airport"}
])

# 增量更新
changes = client.incremental(last_sync="2026-03-16T00:00:00Z")
```
