## 产品定位

Forge 是 Mistral AI 推出的**企业级定制模型训练系统**，允许企业基于自有数据训练前沿 AI 模型。

### 核心问题
- 通用 AI 模型基于公开数据训练，缺乏企业特定知识
- 企业拥有大量内部知识：工程标准、合规政策、代码库、运营流程
- RAG 方案在深度理解方面有天然局限

### Forge 解决方案
将企业知识**内化到模型权重**中，而非仅作为外部检索上下文。

## 技术能力

### 三阶段训练支持

| 阶段 | 说明 | 适用场景 |
|------|------|----------|
| **Pre-training** | 大规模内部数据学习 | 领域感知基础模型 |
| **Post-training** | 特定任务行为微调 | 精细化控制 |
| **Reinforcement Learning** | 对齐内部政策和目标 | Agent 行为优化 |

### RL 的重点应用
- 复杂编排 (orchestration)
- 工具使用 (tool use)
- 决策制定 (decision-making)
- 与内部评估标准对齐

## 战略分析

### Forge vs RAG 路线之争

| 维度 | Forge (微调) | RAG (检索增强) |
|------|-------------|---------------|
| 知识深度 | 深 (内化到权重) | 浅 (上下文注入) |
| 更新速度 | 慢 (需重新训练) | 快 (更新文档即可) |
| 成本 | 高 (GPU 训练) | 低 (向量数据库) |
| 准确性 | 高 (专业领域) | 中 (检索质量依赖) |
| 控制权 | 完全 (自有模型) | 部分 (依赖 API) |

### 市场信号
1. **企业 AI 从 API 消费 → 自有模型**: 数据主权驱动
2. **RAG 天花板**: 企业发现 RAG 在深度理解方面不够
3. **RL for Agents**: 强化学习成为 Agent 行为对齐的标配
4. **战略自主权**: 欧洲企业特别重视 AI 主权

### 与 OpenAI IPO 消息的对比
- OpenAI 转向 IPO → 更商业化、更封闭
- Mistral 推出 Forge → 企业可拥有自己的模型
- 趋势：**AI 主权** vs **AI 服务化** 的路线之争
