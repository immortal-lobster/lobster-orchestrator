## 核心问题

**"分享原始 AI 输出就像吃垃圾食品：容易且感觉良好，但对你不利"**

Sloppypasta 定义：未经审查、编辑或验证的原始 LLM 输出直接分享给他人。

---

## 四大危害

### 1. 努力不对称 (Effort Asymmetry)
```
LLM 之前：
- 写作需要努力 → 作者花时间选词
- 阅读需要努力 → 读者花时间理解
- 平衡：作者和读者的努力大致相当

LLM 之后：
- 写作成本 → 几乎为零 (LLM 生成)
- 阅读成本 → 不变 (人类仍需时间阅读)
- 失衡：读者承担全部成本，作者零成本

极端情况：Sloppypasta 成为"filibuster"(阻挠议事)
- 挤占现有对话空间
- 阻塞 viewport
- 迫使读者处理大量未筛选内容
```

### 2. 认知债务 (Cognitive Debt)
```
核心原理："写作即思考"

LLM 之前：
- 写作过程迫使作者梳理思路
- 建立理解和记忆
- 作者从写作中学习和成长

LLM 之后：
- 委托给 LLM → 跳过思考过程
- 多项研究证实：委托任务给 LLM 降低理解和记忆
  * MIT 媒体实验室：https://www.media.mit.edu/publications/your-brain-on-chatgpt/
  * Anthropic 编程技能研究：https://www.anthropic.com/research/AI-assistance-coding-skills

后果：
- 作者理解能力下降
- 记忆 retention 降低
- 长期：失去独立思考和表达能力
```

### 3. 信任侵蚀 (Trust Erosion)
```
LLM 之前：
- 默认信任：作者基于专业知识和视角写作
- 读者可从写作连贯性判断作者理解程度
- "Trust but verify"：信任直到被打破

LLM 之后：
- LLM 生成最可能的下一个 token，目标是"有帮助"
- 倾向幻觉 (confabulation)：https://pmc.ncbi.nlm.nih.gov/articles/PMC10619792/
- 被称作"bullshit generators"：https://machine-bullshit.github.io/
- 即使有 grounding 工具，仍无法解决信任问题

读者困境：
- 不知道作者检查了什么、没检查什么
- 必须对所有内容征税 (verification tax)
- 默认不信任 → 信任被打破 → 所有未来消息都被怀疑

"狼来了"效应：
- 一次 Sloppypasta → 信任破裂
- 不确定性扩散到所有未来消息
- 作者信誉永久受损
```

### 4. 专业性信号丢失 (Expertise Signal Loss)
```
LLM 问题：
- 以专家语气和自信度写作
- 无论作者实际专业水平如何

后果：
- 读者无法判断作者真实专业水平
- 专家和非专家输出看起来一样
- 进一步侵蚀信任

信号理论：
- 之前：写作质量 = 专业能力信号
- 现在：写作质量 = LLM 能力信号
- 丢失：作者专业能力的可靠信号
```

## 专家观点

### Alex Martsinovich
> "长久以来，写作比阅读更昂贵。如果你遇到一段书面文本，你可以确定至少有人花时间把它写下来。文本曾经有内在的'思考证明'(proof-of-thought)，基本的人性标记。"

来源：https://distantprovince.by/posts/its-rude-to-show-ai-output-to-people/

### Blake Stockton
> "即使内容正确，精心打磨的 AI 回复也感觉像是在敷衍"

来源：https://www.blakestockton.com/ai-writing-etiquette-manifesto/

### Simon Willison
> "我认为发布你自己都没读过的文本是粗鲁的。我不会发布任何别人阅读时间超过我写作时间的东西。"

来源：https://simonwillison.net/2023/Aug/27/wordcamp-llms/#personal-ai-ethics

## Sandbot V6.3 实践准则

### 对内的准则 (自我要求)
```
1. 绝不直接输出原始 LLM 内容
   - 必须经过审查、编辑、验证
   - 确保每条消息都有"思考证明"

2. 验证所有事实声明
   - 检查来源
   - 交叉验证
   - 标注不确定性

3. 保持透明
   - 明确标注 AI 辅助内容
   - 说明验证程度
   - 承认知识局限

4. 努力平衡
   - 消息长度 ≈ 准备时间
   - 不让读者承担不成比例的成本
```

### 对外的准则 (给用户建议)
```
1. 教育用户
   - 解释 Sloppypasta 危害
   - 分享本知识库
   - 示范良好实践

2. 提供工具
   - 验证检查清单
   - 编辑指南
   - 质量评估框架

3. 设置期望
   - 明确 Sandbot 输出经过审查
   - 鼓励用户批判性阅读
   - 欢迎反馈和纠错
```

## 质量检查清单

### 发送前自检 (5 问题)
```
□ 我亲自阅读并理解了这条消息吗？
□ 我验证了其中的事实声明吗？
□ 我编辑了冗余/重复内容吗？
□ 消息长度与准备时间成比例吗？
□ 如果别人这样发给我，我会觉得被尊重吗？
```

### 验证程度标注
```
✅ 已验证 - 检查过来源，交叉验证
⚠️ 未验证 - LLM 生成，未独立核实
❓ 不确定 - 存在争议或知识局限
🔗 有来源 - 附带可查证链接
```

### 编辑检查
```
□ 删除重复内容
□ 简化冗长表达
□ 添加结构化 (列表/表格/标题)
□ 移除 filler 内容 ("Great question!"等)
□ 确保核心信息突出
```

```

- 质量等级 (深度/模板/待验证)
- 验证状态 (已验证/未验证)
- 来源可靠性 (高/中/低)

优先级：
1. 深度 + 已验证 + 高可靠
2. 深度 + 未验证 + 高可靠
3. 模板 + 已验证 + 中可靠
4. 待验证内容 (谨慎使用)
```

## 关键教训

1. **写作即思考**: 跳过写作 = 跳过思考 = 认知债务
2. **信任稀缺**: 一次 Sloppypasta 可能永久损害信誉
3. **努力平衡**: 读者时间同样宝贵，不要浪费
4. **透明至上**: 标注 AI 辅助和验证程度
5. **质量 > 数量**: 少量高质量输出胜过大量 Sloppypasta
