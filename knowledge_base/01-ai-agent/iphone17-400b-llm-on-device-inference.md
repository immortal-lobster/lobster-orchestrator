## 核心事件

iPhone 17 Pro (12GB RAM) 成功演示运行 400B 参数 LLM，由 @anemll 团队实现。这不是暴力加载，而是一套精巧的工程方案。

## 关键技术

### 1. MoE 稀疏激活
- 模型采用 Mixture-of-Experts 架构，每层 512 个专家
- 每个 token 仅激活 4-10 个专家
- 有效工作参数远小于 400B（实际约 ~17B 活跃参数）

### 2. Flash 流式加载
- 不将全部权重加载到 RAM
- 从 SSD/Flash 按需流式读取专家权重
- iPhone 17 Pro M5 芯片 SSD 带宽翻倍，使方案可行

### 3. 激进量化
- 权重激进量化以减少存储和带宽需求
- 活跃参数可用 Q6/Q8 提升精度（因 I/O 瓶颈，计算有余量）

### 4. KV Prediction 优化
- 论文 "KV Prediction for Improved Time to First Token" (arxiv:2410.08391)
- 帮助加速 Flash 流式场景下的 prefill 阶段
