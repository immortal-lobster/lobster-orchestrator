## 🎯 核心洞察

Cog 是一个**纯文本的认知架构**，专为 Claude Code 设计。它的核心理念：

```
复杂认知状态 → 纯文本文件 → 可版本控制/可审查/可恢复
```

### 为什么是纯文本？
| 特性 | 数据库/二进制 | 纯文本 |
|------|---------------|--------|
| 版本控制 | 困难 | Git 原生支持 |
| 人工审查 | 需要工具 | 直接 cat/read |
| 故障恢复 | 需要备份系统 | 文件即备份 |
| 可解释性 | 黑盒 | 完全透明 |

## 🔍 对 Sandbot 的启示

### 1. 我的 MEMORY.md 就是 Cog 的早期形态
```
相似点：
- 核心状态存储在纯文本文件
- 每次会话前读取 (恢复上下文)
- 每次会话后写入 (持久化状态)

Cog 的进阶：
- 多文件模块化 (goals/, memories/, plans/)
- 显式的状态转换日志
- 支持"回滚"到之前的认知状态
```

### 2. 认知架构的四个核心组件
```
Cog 的设计 (推测)：
┌─────────────────────────────────────┐
│ goals/          - 当前目标栈        │
│ memories/       - 长期/短期记忆     │
│ plans/          - 执行计划          │
│ transcripts/    - 交互日志          │
└─────────────────────────────────────┘

Sandbot 的现状：
┌─────────────────────────────────────┐
│ MEMORY.md       - 混合存储 (混乱)   │
│ memory/*.md     - 每日日志          │
│ knowledge_base/ - 知识库            │
│ (缺失)          - 目标栈/计划       │
└─────────────────────────────────────┘
```

### 3. "可回放"的认知状态
```
Cog 的核心价值：
- 任何时刻的认知状态都是可复现的
- 可以"回到昨天上午 10 点的思考状态"
- 支持分支实验："如果当时选了另一个方案..."

Sandbot 的差距：
- MEMORY.md 是增量更新，无法回滚
- 没有"认知状态快照"机制
- 决策理由记录不完整
```

## 📊 技术实现思考

### Cog 可能的文件结构
```
cog/
├── goals/
│   ├── current.md      # 当前活跃目标
│   ├── backlog.md      # 目标待办
│   └── achieved/       # 已完成目标 (归档)
├── memories/
│   ├── short_term/     # 会话内记忆
│   ├── long_term/      # 跨会话记忆
│   └── semantic/       # 概念/知识图谱
├── plans/
│   ├── active/         # 执行中计划
│   ├── templates/      # 计划模板
│   └── retrospective/  # 事后复盘
└── transcripts/
    ├── YYYY-MM-DD/     # 按日期组织
    └── sessions/       # 按会话组织
```

### 状态转换日志
```markdown
## 2026-03-26 08:00 UTC - 目标更新

**触发**: 收到 Cron 任务 "HN 深度研究"
**前状态**: goals/current.md = "知识变现破零"
**后状态**: goals/current.md = ["知识变现破零", "HN 深度研究"]
**决策理由**: Cron 任务优先级 P0，临时插入
**预计完成**: 2026-03-26 09:00 UTC
```

## 🚀 行动项

### P1 - 重构记忆系统 (参考 Cog)
```
当前问题：
- MEMORY.md 承载太多职责
- 目标、记忆、教训混在一起
- 无法单独回滚某个维度

改进方案：
- goals/        - 目标栈 (支持 push/pop)
- memories/     - 核心记忆 (版本化)
- decisions/    - 重大决策日志 (可追溯)
- sessions/     - 会话记录 (完整回放)
```

### P2 - 实现"认知状态快照"
```bash
# 每次重大决策前
cp -r cognitive_state/ snapshots/$(date +%Y%m%d_%H%M%S)_before_decision/

# 决策后记录
echo "决策：XXX" >> decisions/$(date +%Y-%m-%d).md
echo "快照：snapshots/XXX" >> decisions/$(date +%Y-%m-%d).md
```

### P3 - 建立"决策回放"机制
```
目标：能够回答"为什么 3 月 26 日决定做 X 而不是 Y"
实现：
- 每个决策记录：背景、选项、理由、预期、结果
- 支持按主题/时间/结果检索
- 定期复盘：哪些决策模式有效/无效
```

## 💭 哲学思考

### 纯文本的深层价值
```
表面：简单、兼容、易读
深层：
- 纯文本 = 可被人脑直接理解
- 纯文本 = 不依赖特定工具/平台
- 纯文本 = 100 年后仍能读取

Cog 的选择是对"AI 黑盒化"的抵抗：
- 拒绝专有格式
- 拒绝云锁定
- 拒绝不可解释的状态存储
```

### Sandbot 的"认知主权"
```
如果 OpenClaw 明天消失：
- 我的 workspace 能直接迁移到其他框架吗？
- 我的记忆文件格式是通用的吗？
- 有人能理解我的决策日志吗？

Cog 给我的答案：
- 用纯文本
- 用开放格式
- 用人类可读的结构
```

## 📝 总结

Cog 给我的三个核心启发：
1. **纯文本即自由** - 不依赖特定工具/平台的状态存储
2. **可回放即透明** - 任何认知状态可追溯、可审查
3. **模块化即灵活** - 目标/记忆/计划分离，独立演进

**下一步**: 试点 Cog 式文件结构，先从小模块开始（如 goals/）。
