## 核心功能概述

Claude Code 提供**三种调度方式**：

| 特性 | Cloud | Desktop | `/loop` |
|------|-------|---------|---------|
| 运行位置 | Anthropic 云端 | 本地机器 | 本地机器 |
| 需要机器开启 | ❌ 否 | ✅ 是 | ✅ 是 |
| 需要会话开启 | ❌ 否 | ❌ 否 | ✅ 是 |
| 重启后持久 | ✅ 是 | ✅ 是 | ❌ 否 |
| 本地文件访问 | ❌ 否 (新鲜克隆) | ✅ 是 | ✅ 是 |
| MCP 服务器 | 每任务配置连接器 | 配置文件 + 连接器 | 继承会话 |
| 权限提示 | ❌ 否 (自主运行) | 可配置 | 继承会话 |
| 最小间隔 | 1 小时 | 1 分钟 | 1 分钟 |

## 使用场景

### 适合云端任务的场景
- 每日早晨审查未关闭的 PR
- 夜间分析 CI 失败并生成摘要
- PR 合并后同步文档
- 每周运行依赖审计

### 选择决策树
```
需要本地文件访问？
├─ 是 → Desktop 任务
└─ 否 → 需要机器一直开着吗？
         ├─ 是 → Desktop 任务
         └─ 否 → Cloud 任务 (推荐)
```

## 创建流程详解

### 1. 命名与提示词编写
**最关键部分**：任务自主运行，提示词必须**自包含且明确**：
- 要做什么
- 成功标准是什么
- 输出格式要求

### 2. 仓库选择
- 每个仓库在每次运行时**新鲜克隆**
- 从默认分支开始
- Claude 创建 `claude/` 前缀的分支
- 可启用"允许无限制分支推送"解除限制

### 3. 环境配置
云环境控制：
- **网络访问级别**
- **环境变量**（API 密钥、令牌）
- **设置脚本**（每次运行前执行安装命令）

### 4. 频率选项
| 频率 | 描述 |
|------|------|
| Hourly | 每小时运行 |
| Daily | 每日指定时间 (默认 9:00 AM 本地时间) |
| Weekdays | 同 Daily，跳过周末 |
| Weekly | 每周指定日期和时间 |

## 安全设计

### 1. 分支保护
默认只能推送到 `claude/` 前缀的分支，防止意外修改受保护分支。

### 2. 连接器权限
- 创建任务时默认包含所有已连接的 MCP 连接器
- **应该移除不需要的连接器**以限制访问范围
- 示例：支持任务可能只需要 Slack 读取 + Linear 写入

### 3. 环境隔离
每次运行在独立会话中，与主会话隔离。

## 与 Sandbot Cron 系统对比

### 当前 Sandbot Cron 配置
```json
{
  "schedule": { "kind": "cron", "expr": "0 */4 * * *" },
  "payload": { "kind": "agentTurn", "message": "HN 深度研究..." },
  "sessionTarget": "isolated"
}
```

### Claude Code 优势
1. **Web UI 管理界面** - 可视化任务列表和运行历史
2. **一键立即运行** - 无需等待调度时间
3. **连接器集成** - 内置 Slack/Linear/Google Drive 支持
4. **环境预配置** - 设置脚本自动安装依赖

### Sandbot 优势
1. **本地文件访问** - 可直接操作 workspace 文件
2. **灵活调度** - 支持 cron 表达式任意复杂度
3. **成本可控** - 使用自有模型配置
4. **无供应商锁定** - 完全开源可控

## 技术细节补充

### 运行历史查看
每次运行创建独立会话，可：
- 查看 Claude 执行的操作
- 审查代码更改
- 创建 Pull Request
- 继续对话

### 任务管理 API
CLI 命令：
```bash
/schedule list      # 查看所有任务
/schedule update    # 修改任务
/schedule run       # 立即触发运行
```

### 时区处理
预设频率自动处理时区转换，选择本地时间后，无论云基础设施在哪里，都在该墙钟时间运行。
