## 🎯 为什么本地语音助手？

### 隐私驱动
```
云端助手问题：
  - 所有语音上传到服务器 (Amazon/Google/Apple)
  - 意外唤醒也会被记录
  - 数据可能被用于广告定向
  - 政府/第三方访问风险

本地方案优势：
  - 语音数据不出家门
  - 完全控制数据存储
  - 无意外监控
```

### 可靠性驱动
```
云端依赖问题：
  - 网络中断 = 助手瘫痪
  - API 变更可能破坏功能
  - 服务关闭风险 (Google Assistant 已关闭部分功能)
  - 延迟问题 (往返云端)

本地方案优势：
  - 断网仍可用
  - 无 API 依赖
  - 响应延迟更低 (本地处理)
```

### 成本驱动
```
云端成本：
  - 硬件：$50-$200/设备
  - 订阅：部分功能需付费
  - 隐性成本：隐私、数据

本地成本：
  - 硬件：$100-$500 (一次性)
  - 软件：开源免费
  - 电力：~$5-10/年
```

## 🛠️ 推荐技术栈 (2026)

### 唤醒词检测 (Wake Word)
| 方案 | 优点 | 缺点 | 资源占用 |
|------|------|------|----------|
| **Porcupine** | 准确率高、多语言 | 商业许可限制 | ~50MB RAM |
| **Snowboy** | 开源、可自定义 | 维护停止 | ~30MB RAM |
| **Mycroft Precise** | 开源、活跃维护 | 需要训练 | ~100MB RAM |

**推荐**: Porcupine (个人使用免费) 或 Precise (完全开源)

### 语音识别 (STT)
| 方案 | 优点 | 缺点 | 资源占用 |
|------|------|------|----------|
| **Whisper (local)** | 准确率顶级、多语言 | GPU 加速更佳 | 500MB-2GB RAM |
| **Vosk** | 轻量、离线、多语言 | 准确率稍低 | ~200MB RAM |
| **Coqui STT** | 开源、可训练 | 需要调优 | ~300MB RAM |

**推荐**: Whisper (有 GPU) 或 Vosk (资源受限)

### 意图理解 (NLU)
| 方案 | 优点 | 缺点 |
|------|------|------|
| **Home Assistant** | 成熟、集成好 | 学习曲线 |
| **Rasa** | 灵活、可训练 | 需要数据 |
| **自定义规则** | 简单、可控 | 扩展性差 |

**推荐**: Home Assistant (智能家居) 或 Rasa (通用对话)

### 语音合成 (TTS)
| 方案 | 优点 | 缺点 | 资源占用 |
|------|------|------|----------|
| **Piper** | 快速、自然、本地 | 声音选择有限 | ~100MB RAM |
| **Coqui TTS** | 高质量、可训练 | 较慢 | ~500MB RAM |
| **espeak-ng** | 超轻量、多语言 | 机械音 | ~20MB RAM |

**推荐**: Piper (平衡质量和性能)

## 💻 硬件推荐

### 入门级 ($100-200)
```
设备：Raspberry Pi 4 (4GB)
存储：32GB microSD
音频：USB 麦克风 + 3.5mm 音箱
总成本：~$150

性能：
  - 可运行 Vosk + Piper
  - 响应时间：2-5 秒
  - 适合简单命令
```

### 进阶级 ($300-500)
```
设备：Intel NUC / Mini PC
CPU：Intel i5 (8 代+)
内存：16GB RAM
存储：256GB SSD
音频：ReSpeaker 麦克风阵列
总成本：~$400

性能：
  - 可运行 Whisper + Piper
  - 响应时间：1-2 秒
  - 支持多房间音频
```

### 高端级 ($500+)
```
设备：带 GPU 的 Mini PC
GPU：NVIDIA GTX 1650+
内存：32GB RAM
存储：512GB NVMe SSD
音频：专业麦克风阵列 + 功放
总成本：~$700

性能：
  - Whisper 大模型 + GPU 加速
  - 响应时间：<1 秒
  - 多并发用户支持
```

```bash
# 使用 Home Assistant OS (推荐)
```

### 步骤 2: 安装语音插件
```yaml
# configuration.yaml
wake_word:
  - platform: porcupine
    access_key: YOUR_KEY
    keywords:
      - hey jarvis

stt:
  - platform: whisper
    model: medium

tts:
  - platform: piper
    voice: en_US-lessac-medium
```

### 步骤 3: 配置音频硬件
```bash
# 测试麦克风
arecord -l
arecord -f cd -d 5 test.wav

# 测试音箱
aplay -l
speaker-test -t wav

# 调整音量
alsamixer
```

### 步骤 4: 创建自动化
```yaml
# automations.yaml
- alias: "Turn on lights"
  trigger:
    platform: voice
    command: "turn on the lights"
  action:
    entity_id: light.living_room
    service: light.turn_on
```

## ⚠️ 常见陷阱

### 1. 麦克风选择
```
❌ 错误：使用笔记本内置麦克风
   - 距离远、噪音大、唤醒率低

✅ 正确：使用麦克风阵列
   - 波束成形、降噪、远场识别
   - 推荐：ReSpeaker 4-Mic Array
```

### 2. 网络配置
```
❌ 错误：依赖 DHCP 动态 IP
   - IP 变化导致连接中断

✅ 正确：配置静态 IP 或 DNS 保留
   - 确保设备始终可访问
```

### 3. 资源不足
```
❌ 错误：在 Pi Zero 上运行 Whisper
   - 响应时间>10 秒，体验极差

✅ 正确：根据硬件选择模型
   - Pi 4: Vosk 或 Whisper tiny
   - NUC+: Whisper medium/large
```

### 4. 忽略隐私
```
❌ 错误：日志记录所有语音
   - 占用存储、隐私风险

✅ 正确：仅记录命令和错误
   - 定期清理日志
   - 加密存储
```

## 💡 进阶技巧

### 1. 多房间音频
```
方案：使用 ESP32 + Audio 板
成本：~$20/房间
同步：使用 Snapcast 或 AirPlay 2
效果：全屋同步播放
```

### 2. 离线 LLM 集成
```
工具：Ollama + Llama 3 8B
硬件：16GB+ RAM
用途：复杂对话、推理
延迟：5-15 秒 (可接受)
```

### 3. 自定义唤醒词
```
工具：Porcupine 训练工具
流程：
  1. 录制 50-100 次唤醒词
  2. 训练模型
  3. 部署到设备
注意：避免常见词，减少误触发
```

### 4. 能源优化
```
策略：
  - 空闲时降低 CPU 频率
  - 使用低功耗麦克风 (始终监听)
  - 主处理单元休眠唤醒
效果：功耗从 10W → 2-3W
```
