## 核心问题

全注意力瓶颈限制 LLM 有效上下文到 128K-1M。现有方案（混合线性注意力、固定状态 RNN、RAG/Agent 外部存储）要么精度快速衰减，要么缺乏端到端可微分性，要么需要复杂管道。

## MSA 方案

Memory Sparse Attention (MSA) 是一个端到端可训练、可扩展的稀疏潜在状态记忆框架。

### 三大核心创新

#### 1. 稀疏注意力 + Document-wise RoPE
- 每个文档从位置 0 重新开始，防止训练短/推理长的位置漂移
- 64K 训练可外推到 100M token
- 实现 O(L) 近线性复杂度（训练+推理）

#### 2. KV Cache 压缩 + Memory Parallel 推理引擎
- 文档潜在状态 (K/V/Kᵣ) 通过 chunk-mean pooling 压缩
- Router projector 用余弦相似度计算相关性
- Top-k 文档选择，拼接压缩 K/V 用于自回归解码
- 2×A800 GPU 即可实现 100M token 吞吐

#### 3. Memory Interleave 多轮多跳推理
- 跨分散记忆片段的多轮、多跳推理
- 路由仅应用于上层；下层保持独立文档处理实现层次对齐

### 三阶段管道
1. **Global Memory Encoding (离线)**: 前向传播语料库，缓存 chunk-pooled (K̄, V̄, K̄ᵣ)
2. **Online Query Routing**: 实时路由查询到相关文档
3. **Autoregressive Decoding**: 拼接检索的压缩 K/V 进行生成

## 性能数据

| 指标 | MSA | 对比 |
|------|-----|------|
| Token 范围 | 16K → 100M | 同 backbone RAG 最大 ~1M |
| 退化率 | <9% (全范围) | RAG 随长度快速退化 |
| 硬件需求 | 2×A800 | 竞品需要更多 |
| 复杂度 | O(L) | 全注意力 O(L²) |
| NIAH 基准 | 超越 RAG 和长上下文模型 | - |

## 技术意义

### 解耦记忆容量与推理
- 这是 MSA 最核心的贡献：记忆容量不再受推理窗口限制
- 端到端可微分，无需外部检索管道
- 将检索和生成融合到单一可微分循环中

### 与 RAG 的对比
| 维度 | MSA | RAG |
|------|-----|-----|
| 可微分 | ✅ 端到端 | ❌ 检索不可微 |
| 管道复杂度 | 低 (单模型) | 高 (检索+生成) |
| 长上下文 | 100M+ | 通常 <1M |
| 精度保持 | <9% 退化 | 随长度退化 |

## 对 AI Agent 的影响

1. **超长上下文 Agent**: 100M token 记忆 = Agent 可处理整个代码库/文档库
2. **去 RAG 化**: 减少 Agent 架构复杂度
3. **多轮记忆**: Memory Interleave 天然适合 Agent 多轮对话
4. **边缘部署**: O(L) 复杂度降低硬件门槛
