## 📊 核心故事

一位白领会销售顾问想为害虫控制行业构建 Vertical SaaS，决定**亲自应聘成为一线技术员**进行田野调查。13 天打破公司记录获得执照，21 天内签下$24k 年单，最终决定**自己收购小公司创业**而非卖 SaaS 给现有企业。

---

## 🔍 关键洞察

### 1. 田野调查的价值
```
传统方式：
- 请求跟车访问 → 被拒绝太多
- 调研电话 → 没人接
- 行业报告 → 表面数据

他的方式：
- 亲自应聘 → 深入一线
- 实地工作 → 发现真实痛点
- 建立信任 → 获得内部视角
```

### 2. 行业特点分析（害虫控制）
| 特点 | 说明 | SaaS 机会 |
|------|------|----------|
| 经常性收入 | 定期消杀服务 | ✅ 订阅模式天然契合 |
| 专业化 | 需要执照/培训 | ✅ 培训管理系统 |
| 碎片化 | 大量小公司 | ✅ 整合机会 |
| 监管严格 | 化学品管理 | ✅ 合规追踪 |
| TAM | 美国$300 亿 | ✅ 市场够大 |

### 3. 大公司的创新困境
```
他加入的是全国性集团（年收入$B 级别）：
- 核心系统：重度定制的 Salesforce
- 技术员工抱怨：系统难用但"无法替换"
- 培训经理：对他的培训 GPT 不感兴趣
  原因：可能取代他 1/4 的工作
- 销售流程：需要多个签名，差点丢掉$24k 订单

核心问题：
"员工不想掀翻船，没有改进激励。
待在舒适区更安全。"
```

### 4. 一线员工的隐藏价值
```
他跟资深技术员时，与客户聊天做了小额追加销售：
- 没有接受过销售培训
- 但"脚踏实地"时自然发现机会
- 结果：被提供销售岗位

洞察：
- 一线员工最了解客户痛点
- 但现有系统不激励他们贡献
- 这是 SaaS 可以赋能的点
```

## 🎯 创业决策

### 为什么不卖 SaaS 给现有公司？
```
1. 创新阻力大
   - 员工不想改变
   - 管理层不愿冒险
   - "能用就行"心态

2. 销售周期长
   - 需要多个决策者
   - 定制化需求多
   - 价格敏感

3. 价值难量化
   - 现有系统"还能用"
   - ROI 不清晰
   - 替换成本高
```

### 为什么选择收购创业？
```
1. 从零开始塑造文化
   -  hire 认同理念的人
   -  没有历史包袱
   -  快速迭代

2. 垂直整合
   -  自己运营 → 自己开发工具
   -  产品即服务
   -  真实场景验证

3. 可扩展性
   -  证明模式后可复制
   -  平台化增长
   -  收购更多小公司
```
