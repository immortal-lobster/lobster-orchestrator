### A10-201: RPA 机器人流程自动化

**定义**: RPA 是用软件机器人自动执行重复、规则明确的业务流程的技术

**核心原理**: 
- 屏幕抓取：识别 UI 元素并模拟操作
- 工作流编排：定义任务执行顺序
- 触发器：时间/事件/数据驱动执行
- 异常处理：规则引擎处理边界情况

**实际案例**: 
- Sandbot 心跳系统：Cron 每 30 分钟自动执行知识获取、趋势扫描、深度学习任务
- 已连续运行 105 轮，100% 成功率
- Lobster Orchestrator：自动管理 50 个 PicoClaw 实例，<10MB/实例内存
- 批量互动脚本：自动点赞/评论/发帖，安全防限流

**关键数据**: 
- 人力节省：80-95% (重复任务)
- 错误率降低：从 5% → <0.1%
- ROI: 200-500% (6-12 个月回本)
- Sandbot 日调用：从~5000 次 → ≤200 次 (成本优化 96%)

**应用场景**: 
- 数据录入和迁移
- 报表生成和分发
- 跨系统数据同步
- 定时任务调度 (Cron 增强)

**配置参数**: 
- 执行频率：每分钟 - 每天
- 并发机器人：1-100 个
- 超时设置：30 秒 -1 小时
- 重试策略：1-5 次，指数退避

**扩展阅读**: 
- UiPath 官方文档
- 《Robotic Process Automation》Apress
- Gartner RPA 魔力象限报告

### A10-202: 工作流引擎架构

**定义**: 工作流引擎是编排、执行和监控业务流程的中间件系统

**核心原理**: 
- BPMN 2.0: 标准业务流程建模符号
- 状态机：明确定义状态和转换
- 任务调度：优先级队列、负载均衡
- 持久化：流程状态数据库存储
- 可观测性：日志、指标、追踪

**实际案例**: 
- Sandbot 联邦智能：7 子 Agent 通过工作流协作
  - 主 Agent 接收任务 → 分析类型 → 分配子 Agent
  - Auditor 质量审查 → 主 Agent 最终交付
- Cron 作业系统：
  - #105:  Lobster Orchestrator 每日 Git 同步
  - #106: 成本监控脚本 (每日 200 次调用上限)
  - #107: 不死龙虾联盟互动

**关键数据**: 
- 吞吐量：1000+ 工作流实例/秒
- 延迟：P99 <100ms (简单流程)
- 可用性：99.9%+ (集群部署)
- MTTR: <5 分钟 (自动恢复)

**应用场景**: 
- 审批流程 (采购、报销)
- 数据管道 (ETL/ELT)
- CI/CD流水线
- 客户服务工单流转

**配置参数**: 
- 最大并发：10-10000 实例
- 超时策略：5 分钟 -24 小时
- 持久化间隔：每步/每批次
- 重试次数：0-10 次

**扩展阅读**: 
- Camunda 工作流引擎文档
- 《Workflow Patterns》MIT Press
- AWS Step Functions 最佳实践
- Apache Airflow 官方教程
