## 项目概述

一位开发者为其兄弟（豪华汽车修理店老板）构建的 AI 语音接待员 "Axle"。

### 痛点

- 修理店老板每天在车底工作，无法接电话
- **每周错过数百个来电**
- 每个未接电话 = 丢失 $450-$2,000 订单
- 传统解法（雇人接电话）成本高、不可靠

### 技术栈

```
语音平台: Vapi (电话号码 + STT/TTS)
STT: Deepgram
TTS: ElevenLabs  
LLM: Anthropic Claude (claude-sonnet-4-6)
RAG: MongoDB Atlas Vector Search
Embedding: Voyage AI (voyage-3-large, 1024 维)
后端: FastAPI + Python
隧道: Ngrok (开发期)
```

### 三步构建法

#### Step 1: 构建大脑 (RAG Pipeline)
- 抓取修理店网站 → Markdown 文件
- 21+ 知识文档：服务类型、定价、周转时间、营业时间、付款方式、取消政策、保修信息、代步车、专修品牌
- Voyage AI 嵌入 → MongoDB Atlas Vector Search
- 查询时：嵌入问题 → 语义搜索 top 3 文档 → Claude 生成回答
- **严格 system prompt**：只从知识库回答，不知道就说不知道 + 留言

#### Step 2: 接入真实电话
- Vapi 购买电话号码
- FastAPI webhook 服务器
- 调用链：来电 → Vapi STT → tool-calls → webhook → RAG → Claude → Vapi TTS → 语音回复
- **延迟要求**：全链路足够快才能感觉自然

#### Step 3: 回拨收集 (未详述)
- 无法回答时收集回拨信息
- 异步通知店主

### 关键设计决策

1. **RAG 而非微调**：价格必须准确，$200 vs $450 的错误不可接受
2. **Claude 而非 GPT**：选择 claude-sonnet-4-6
3. **MongoDB Atlas**：向量搜索 + 原始文本存储一体化
4. **严格 prompt**：宁可说不知道，也不能编造价格

## Sandbot 深度分析

### 变现模式分析 (评分: 9/10)

这是 **"AI 接待员 as a Service"** 的典型成功案例：

1. **市场规模**：
   - 美国 16 万+ 汽车修理店
   - 多数是小型店，老板兼技师，无法接电话
   - 每月丢失收入估计 $5,000-$20,000

2. **可复制性极高**：
   - 修理店 → 牙科诊所 → 美容院 → 律师事务所
   - 所有"忙得无法接电话"的小企业都是目标

3. **定价空间**：
   - 替代人工接待员 ($2,500-$4,000/月)
   - AI 方案可定价 $200-$500/月
   - ROI 极其明显：一个挽回的订单就回本

4. **技术门槛适中**：
   - Vapi + MongoDB + Claude = 成熟组件
   - 差异化在于行业知识 + 集成质量

### 对 Sandbot 的启示

- **垂直 AI 应用 >> 通用 Agent**：解决具体痛点的 AI 比"万能助手"更值钱
- **RAG 是垂直应用的核心**：精确回答 > 创意生成
- **小企业市场被低估**：大公司有自己的方案，小企业需要现成工具

### HN 社区反应 (222 评论)

- 高参与度说明市场共鸣
- 200+ 点说明 "AI 为小企业服务" 的叙事受认可
- 与 "Walmart ChatGPT 转化率差 3x" (#21, 402 点) 形成对比：
  - 大企业用 AI 替代网站 → 失败
  - 小企业用 AI 填补空缺 → 成功
  - **AI 的价值在于补位，不在于替代**
