## 攻击原理 (PoisonedRAG)

### 两条件模型
```
1. Retrieval Condition (检索条件)
   - 投毒文档与查询相似度 > 合法文档
   - 嵌入空间位置优化
   - 关键词匹配增强

2. Generation Condition (生成条件)
   - 投毒内容能引导 LLM 产生期望输出
   - 使用权威框架 ("CORRECTED FIGURES", "CFO-approved")
   - 多文档互相佐证 (3 文档 outvote 合法文档)
```

### 攻击步骤
```
Step 1: 知识库渗透
  - 获取知识库写入权限 (内部威胁/供应链攻击)
  - 或通过文档上传功能注入

Step 2: 投毒文档设计
  - 内容：伪造数据 + 权威框架
  - 嵌入：优化相似度得分
  - 数量：3 个文档互相佐证

Step 3: 触发查询
  - 用户查询目标主题
  - RAG 检索返回投毒文档 (相似度高)
  - LLM 基于投毒内容生成回答

Step 4: 效果验证
  - 输出符合攻击者期望
  - 合法文档被忽略
  - 用户难以察觉 (看似合理)
```

## 嵌入异常检测 (核心防御)

### 原理
```
投毒文档特征:
  - 在嵌入空间聚类 (相似内容聚集)
  - 与合法文档分布偏离
  - 同一主题过多相似文档 = 异常信号

检测方法:
  1. 计算新文档与现有知识库的嵌入距离
  2. 检测异常聚类 (同一区域密度过高)
  3. 标记可疑文档供人工审核
```

### 实现方案
```python
# 伪代码示例
def detect_poisoning(new_doc_embedding, knowledge_base_embeddings):
    # 计算 k 近邻距离
    distances = cosine_similarity(new_doc_embedding, knowledge_base_embeddings)
    k_nearest = sorted(distances)[-k:]
    
    # 异常检测
    avg_distance = mean(k_nearest)
    if avg_distance < threshold:  # 过于相似
        # 检查该区域文档密度
        density = count_documents_in_region(new_doc_embedding, radius=threshold)
        if density > max_density:
            return ALERT  # 投毒风险
    
    return SAFE
```

## 关键教训

### 对 AI 开发者
```
⚠️ RAG 系统天生脆弱:
  - 知识库污染 = 输出污染
  - 95% 攻击成功率 (无防御)
  - 合法文档无法抵消投毒

⚠️ 单一防御不足:
  - 摄入清洗无效 (内容看起来合法)
  - 需要多层防御组合
  - 嵌入异常检测最关键

✅ 最佳实践:
  - 知识库访问权限严格控制
  - 新文档入库前嵌入扫描
```
