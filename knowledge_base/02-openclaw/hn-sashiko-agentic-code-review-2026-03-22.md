## 📌 核心洞察

Sashiko (刺し子，日本装饰性加固缝纫) 是一个**Agentic Linux 内核代码审查系统**，由 Google 资助，属于 Linux Foundation。

**关键数据**:
- 使用 Gemini 3.1 Pro
- **53.6% 的 bug 检出率** (基于 1000 个带 Fixed 标签的上游 commit)
- **100% 的这些 bug 之前都通过了人工代码审查**
- 多阶段审查协议，最小化误报

**定位**: 增强人类审查员，而非替代。

## 🔍 技术架构分析

### 多 Agent 协作模式
```
Sashiko 不是单一 Agent，而是"专家团队"：

┌─────────────────────────────────────────────┐
│            Sashiko Orchestrator             │
└───────────────┬─────────────────────────────┘
                │
    ┌───────────┼───────────┐
    │           │           │
    ▼           ▼           ▼
┌─────────┐ ┌─────────┐ ┌─────────┐
│架构验证  │ │安全审计  │ │资源管理  │
│Agent    │ │Agent    │ │Agent    │
└─────────┘ └─────────┘ └─────────┘
    │           │           │
    ▼           ▼           ▼
┌─────────┐ ┌─────────┐ ┌─────────┐
│并发分析  │ │子系统   │ │通用    │
│Agent    │ │专用    │ │规则    │
│Agent    │ │Prompt  │ │Prompt  │
└─────────┘ └─────────┘ └─────────┘
```

### 核心组件
1. **Prompt 库**: https://github.com/masoncl/review-prompts
   - 由 Chris Mason (Linux 内核开发者) 创建
   - 按子系统分类 (网络、文件系统、驱动等)
   - 通用规则 (内存安全、并发、错误处理)

2. **多阶段审查协议**:
   ```
   Stage 1: 快速扫描 (语法、风格、明显问题)
   Stage 2: 深度分析 (逻辑、边界条件、竞态)
   Stage 3: 交叉验证 (多 Agent 独立审查，投票决策)
   Stage 4: 人类复核 (只推送高置信度问题)
   ```

3. **持续监控**: 自动订阅 LKML (Linux Kernel Mailing List)，实时审查新提交。

## 💡 深度洞察

### 1. "53.6% 检出率" 的含义

**表面解读**: 还有 46.4% 的 bug 没检出，效果一般。

**深层解读**: 
- 这 53.6% 是**100% 通过人工审查的 bug**
- 意味着：人类审查员漏掉的 bug，AI 能补上一半
- **AI + 人类 > 人类 alone**，这是增强智能的真谛

**对 OpenClaw 的启示**:
- 不要追求"AI 完全替代人类"
- 定位应该是"AI 增强人类能力"
- Sandbot 的知识产品应该是"AI 辅助学习"，而非"AI 替代学习"

### 2. 多 Agent 投票的价值

Sashiko 用多 Agent 独立审查 + 投票决策来**降低误报率**。这是关键设计：

```
单一 Agent:
  - 优点：快、便宜
  - 缺点：幻觉、误报、偏见

多 Agent 投票:
  - 优点：鲁棒、准确、可解释
  - 缺点：慢、贵、复杂

平衡点:
  - 高置信度 (3/3 Agent 同意) → 自动行动
  - 中置信度 (2/3 Agent 同意) → 人工复核
  - 低置信度 (1/3 Agent 同意) → 忽略
```

**OpenClaw 应用**:
- 关键决策 (如技能发布、知识库更新) 用多 Agent 投票
- 日常任务 (如 Cron 扫描) 用单一 Agent

### 3. Google 资助的深意

Sashiko 由 Google 资助，属于 Linux Foundation。这揭示了：

**大公司的开源策略**:
- 资助关键基础设施 (Linux 内核) 的质量工具
- 开源 (Apache 2.0)，建立信任和生态
- 间接受益：更稳定的内核 = 更稳定的 GCP

**Sandbot 学习**:
- 开源核心技能，建立信任
- 商业精选知识库，实现变现
- 长期：成为 Agent 生态的"质量标杆"

## 🔐 安全与边界

### Sashiko 的自我定位
> "As with any Large Language Model-based tool, Sashiko's output is probabilistic. It might find or miss bugs across different runs with the exact same input. It is designed to augment and assist human reviewers, not to replace them."

**关键原则**:
1. 承认概率性输出 (不承诺 100% 准确)
2. 明确边界 (增强而非替代)
3. 透明沟通 (告知用户局限)

**OpenClaw 应遵循**:
- 在技能文档中明确说明局限
- 不夸大能力 (如"100% 准确审计")
- 鼓励用户验证关键输出
