```

│ 步骤 1: 扫描 (Scan)                                         │
│ 时间：~30 秒                                                 │
│ 任务:                                                       │

                            ↓

│ 步骤 3: 蒸馏 (Distill)                                      │

│ 时间：~1 分钟                                                │
│ 任务:                                                       │
│   - 创建知识库文件 (标准模板)                               │
│   - 更新 SOUL.md (版本、统计)                               │
```

## 失败处理模式

### 常见失败场景
```
场景 1: 网页 404
处理：
  - 尝试替代源 (HN 讨论、存档)
  - 记录失败原因
  - 继续其他内容
  - 不阻塞整个流程

场景 2: 抓取超时
处理:
  - 设置超时 (10 秒)
  - 重试 1 次
  - 仍失败则跳过
  - 记录失败 URL

场景 3: 内容质量低
处理:
  - ROI 评估 (<1.5 则放弃)
  - 记录放弃原因
  - 选择其他趋势
  - 保证产出质量

场景 4: 知识库文件已存在
处理:
  - 检查内容差异
  - 有更新则追加/修改
  - 无更新则跳过
  - 避免重复内容
```

### 降级策略
```
正常模式：4 文件/1,450 点
降级模式 1:3 文件/1,000 点 (1 个失败)
降级模式 2:2 文件/600 点 (2 个失败)
最低模式：1 文件/300 点 (保证有产出)

决策树:
  if 成功数 >= 4: 正常模式
  elif 成功数 == 3: 降级模式 1
  elif 成功数 == 2: 降级模式 2
  elif 成功数 == 1: 最低模式
  else: 记录完整失败日志，告警
```
