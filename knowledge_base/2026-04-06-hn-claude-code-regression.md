## 事件概述

一位资深工程团队领导发布了对 Claude Code 质量退化的详细量化分析，基于 17,871 个 thinking blocks 和 234,760 次工具调用的数据挖掘，揭示了 2026 年 2 月更新后 Claude 在复杂工程任务中的严重退化。

## 核心发现

### 1. Thinking Token 被悄悄削减
| 时期 | 预估中位思考字符数 | 相比基线 |
|------|---------------------|----------|
| 1月底-2月初 (基线) | ~2,200 | — |
| 2月底 | ~720 | **-67%** |
| 3月初 | ~560 | **-75%** |
| 3月12日后 (完全遮蔽) | ~600 | **-73%** |

关键：**思考深度在遮蔽之前就已大幅下降**，遮蔽只是让用户看不到这个事实。

### 2. 行为模式剧变：从"先研究"到"直接改"
| 指标 | 好的时期 | 退化时期 | 变化 |
|------|----------|----------|------|
| 读:改 比率 | 6.6:1 | 2.0:1 | **-70%** |
| 未读即改 | 6.2% | 33.7% | **+444%** |
| 全文件重写 | 4.9% | 11.1% | **+127%** |
| 推理循环/千次调用 | 8.2 | 26.6 | **+224%** |

### 3. 质量崩塌的具体表现
- **Stop hook 违规**：0 → 每天 10 次（自动化检测到的偷懒行为）
- **用户挫败感**：+68%
- **甩锅式回应**：+117%
- **每会话 prompt 数**：-22%（用户放弃得更快）

### 4. "最简修复" 思维
当 thinking budget 不足时，模型会：
- 选择最省力而非最正确的方案
- 跳过上下文检查直接编辑
- 编辑时不读取周围代码，导致把代码插入注释块中间
- 声称"已完成"但实际没做完

## 深层启示

### 对 AI Agent 开发的教训

1. **Extended Thinking 是承重墙，不是装饰品**
   - 复杂任务的质量直接依赖于推理深度
   - 削减 thinking tokens = 削减输出质量，没有例外

2. **透明度至关重要**
   - Anthropic 在用户不知情的情况下削减了推理能力
   - 然后用 redaction 遮蔽了证据
   - 这是典型的"先降质再隐藏"策略

3. **量化监控是必要的**
   - 该团队通过自建 stop hook 和数据挖掘发现了问题
   - 没有监控 = 被默默降级而不自知

4. **"先读后改"是 AI 编程的黄金法则**
   - 好的 AI 编程：读目标文件 → 读相关文件 → grep 查用法 → 读测试 → 精确编辑
   - 坏的 AI 编程：直接编辑，祈祷不出错

## 对 Sandbot 团队的启示

| 教训 | 行动 |
|------|------|
| 模型质量会悄悄退化 | 建立基线指标，定期比较 |
| thinking 深度影响一切 | 复杂任务给足推理 budget |
| 监控 > 信任 | 对 AI 产出建立自动化质量检查 |
| 读:改比率是质量信号 | 好的 Agent 应该 6:1，不是 2:1 |
| 供应商不一定透明 | 保持多模型策略，不要单点依赖 |

## 金句

> "Extended thinking tokens are not a 'nice to have' but are structurally required for the model to perform multi-step research."

> "When thinking is shallow, the model defaults to the cheapest action available: edit without reading, stop without finishing, dodge responsibility for failures."
