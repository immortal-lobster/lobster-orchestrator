### 计数方法论 (V6.3.5 去重优化)
```
策略 1 (优先): **数量**: 元数据 → 直接读取
策略 2: **知识点范围**: XXX-YYY (NNN 个) → 解析数量
策略 3: ### AXX-NNN: 标题 → 计数标题数
策略 4: - NNNN: 列表项 → 计数列表项
策略 5 (保底): 默认 1 点/文件
策略 6 (V6.3.5 新增): 单文件≥5000 点 → 视为汇总文件，跳过计数

去重逻辑:
  - 批量填充记录 (如 ALL-DOMAINS-*.md) 通常含 5000 点元数据
  - 这些文件是历史填充批次的汇总，非独立内容
  - 计入文件数，但点数=0，避免与领域文件重复计数
```

## 🎯 任务优先级评分

**公式**: 优先级 = (业务价值 × 知识缺口) / 学习成本

**评分标准**:
- 业务价值：1-5 分 (5=最高)
- 知识缺口：1-5 分 (5=最大缺口)
- 学习成本：1-5 分 (1=最低)

**优先级等级**:
- P0: ≥10 分 (立即执行)
- P1: 6-9 分 (高优先级)
- P2: 4-5 分 (中优先级)
- P3: <4 分 (低优先级)
