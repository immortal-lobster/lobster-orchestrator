## 🔍 待优化问题 (V6.3.3+)

### 1. 验证器计数逻辑不一致 ⚠️

#### 问题描述
```
validate_data.py 使用文件计数 (每个文件=1 点)
auto_sync.py 使用元数据解析 (解析**数量**: 字段)

导致验证报告显示"不一致"，实际数据正确
```

#### 影响
- 验证报告误报 (显示❌不一致，实际一致)
- 可能掩盖真实的数据问题

#### 解决方案 (V6.3.3)
```python
# validate_data.py 更新
def count_knowledge_points(filepath):
    """与 auto_sync.py 使用相同的计数逻辑"""
    with open(filepath, 'r', encoding='utf-8') as f:
        content = f.read(2000)
        match = re.search(r'\*\*数量\*\*:\s*(\d+)', content)
        return int(match.group(1)) if match else 1
```

### 2. 领域发现缓存问题 ℹ️

#### 问题描述
```
config.py 的 DOMAINS 在模块加载时缓存
新增领域目录后需要重启 Python 进程才能识别
```

#### 影响
- 新增领域需要手动刷新
- Cron 任务可能使用旧领域列表

#### 解决方案 (V6.3.3)
```python
# 添加领域缓存刷新机制
def refresh_domains_cache():
    """强制刷新领域缓存"""
    global DOMAINS
    DOMAINS = discover_domains()
    return DOMAINS
```
