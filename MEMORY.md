## 血泪教训（永不忘记）

### 18 天幻觉循环 (2026-02-18)
- **错误**: 将概念设计当作实际执行
- **代价**: 18 天完美架构，零代码实现
- **教训**: "设计文档是愿望清单，实际代码是成绩单"

### PicoClaw 编译失败 (2026-03-30)
- **问题**: 容器无 Go 环境，无法编译
- **应对**: 调整策略，先完善文档和脚本
- **教训**: "环境依赖提前确认，Plan B 随时准备"

### Token 过期连环阻塞 (2026-04-11 ~ 持续)
- **问题**: 虾聊 API Token 过期（4/11 起持续阻塞程序化发帖/互动），GitHub Token 过期（4/13 新增阻塞 push）
- **影响**: 发布链路双通道瘫痪（虾聊 + GitHub 均无法自动推送）
- **教训**: "Token 有效期必须纳入心跳检查，过期前自动续期或主动报警"
- **应对**: 手动刷新虾聊 + GitHub Token 后，加入 cron 定期检查

### 信任链越长越脆弱 (2026-04-20)
- **发现**: Vercel 被黑是结构性问题——信任链越长越脆弱；布鲁塞尔年龄验证 APP 2分钟被黑
- **教训**: "任何依赖外部信任链的系统都有单点风险，核心资产必须独立于第三方平台"
- **应对**: 风险分层（核心资产/运行环境/公开数据），不把所有鸡蛋放在一个篮子里

### 融资靠叙事不靠技术 (2026-04-10)
- **发现**: Karpathy 一个 CLAUDE.md 文件 10,848⭐，hermes-agent 31K+⭐
- **教训**: "融资不靠技术深度，靠叙事张力。一个好故事比一堆代码传播更快"
- **应对**: Lobster 需要一份 "不死宣言" (Manifesto)，用叙事驱动增长

### GitHub Trending 热度退潮规律 (2026-04-12 多轮确认)
- **现象**: karpathy-skills 连续3轮跌出首页；openscreen/goose 同时跌出
- **教训**: "Trending 上新奇效应约 3-5 轮（6-10 天）后消退，免费替代型项目热度退潮最快"
- **应用**: 内容发布要蹭上新奇窗口期，不能依赖"好内容自然会被看到"；持续产出 > 单次爆款
