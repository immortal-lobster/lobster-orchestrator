# 🚨 模型调用限制

**版本**: V0.4.2  
**生效时间**: 2026-03-31  
**每日上限**: 200 次

---

## 📊 限制规则

### 基本原则

```
每日调用上限：200 次
超限后：自动暂停 + 报警
恢复：需要用户确认
```

### 监控机制

**监控脚本**: `scripts/model-monitor.sh`  
**执行频率**: 每 10 分钟一次  
**检查内容**:
- 统计当日调用次数
- 检查是否接近上限 (80%)
- 检查是否超过上限 (100%)

---

## 🚨 报警流程

### 阶段 1: 警告 (80%)

**触发条件**: 调用次数 ≥ 160 次  
**动作**:
- ✅ 记录日志
- ✅ 发送警告通知
- ✅ 继续监控

### 阶段 2: 暂停 (100%)

**触发条件**: 调用次数 ≥ 200 次  
**动作**:
- ✅ 立即暂停模型调用
- ✅ 创建警报文件
- ✅ 发送警报通知
- ✅ 等待用户确认

---

## 🔧 实施方式

### 方式 1: Cron 任务 (推荐)

**添加到 crontab**:
```bash
# 每 10 分钟检查一次
*/10 * * * * /home/node/.openclaw/workspace/scripts/model-monitor.sh
```

### 方式 2: Gateway 集成

**Gateway 配置**:
```json
{
  "model": {
    "usageLimit": {
      "daily": 200,
      "action": "pause",
      "notify": true
    }
  }
}
```

### 方式 3: 调用前检查

**在每次调用模型前**:
```bash
# 检查是否暂停
if [[ -f "$WORKSPACE/memory/.model_calls_paused" ]]; then
    echo "❌ 模型调用已暂停，等待用户确认"
    exit 1
fi

# 检查计数
if [[ $(cat .model_call_count_$(date +%Y-%m-%d)) -ge 200 ]]; then
    echo "🚨 已达每日上限"
    exit 1
fi
```

## 📈 监控文件

### 调用计数文件

**位置**: `memory/.model_call_count_YYYY-MM-DD`  
**内容**: 当日调用次数  
**示例**:
```
156
```

### 暂停标志文件

**位置**: `memory/.model_calls_paused`  
**存在时**: 禁止模型调用  
**移除**: 用户确认后删除

### 警报文件

**位置**: `memory/.model_alert`  
**内容**: JSON 格式警报信息  
**示例**:
```json
{
  "alert": "MODEL_CALL_LIMIT_EXCEEDED",
  "count": 200,
  "limit": 200,
  "time": "2026-03-31 17:30:00",
  "status": "PAUSED",
  "message": "模型调用已达每日上限，已暂停调用"
}
```

### 监控日志

**位置**: `memory/model-usage.log`  
**格式**: 时间戳 + 调用次数  
**示例**:
```
2026-03-31 17:00:00 - 当前调用：150/200
2026-03-31 17:10:00 - 当前调用：165/200
2026-03-31 17:20:00 - ⚠️ 模型调用警告：165/200 (80%)
2026-03-31 17:30:00 - 🚨 模型调用超限警告！
```

## 🛠️ 用户操作

### 查看当前调用次数

```bash
cat memory/.model_call_count_$(date +%Y-%m-%d)
```

### 查看监控日志

```bash
tail -20 memory/model-usage.log
```

### 解除暂停 (确认继续)

```bash
# 1. 查看警报
cat memory/.model_alert

# 2. 确认继续 (删除暂停标志)
rm memory/.model_calls_paused

# 3. 重置计数 (可选)
rm memory/.model_call_count_$(date +%Y-%m-%d)

# 4. 清除警报
rm memory/.model_alert
```

### 临时增加上限

```bash
# 修改监控脚本
DAILY_LIMIT=300  # 临时改为 300
```

## 🦞 最佳实践

### ✅ 应该做的

1. **批量操作** - 一次调用完成多个任务
2. **本地执行** - 简单任务不调用模型
3. **监控日志** - 定期检查调用次数
4. **优化提示** - 减少不必要的 tokens

### ❌ 不应该做的

1. **单次单文件** - 每次只写一个小文件
2. **简单任务** - 让模型执行 bash 能做的
3. **频繁心跳** - 每次都调用模型
4. **完整上下文** - 每次都带所有文件
