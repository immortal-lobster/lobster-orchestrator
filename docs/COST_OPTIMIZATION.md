# 💰 成本优化指南

**版本**: V0.4.1  
**目标**: 减少 90% 模型调用，节省 90% 成本

---

## 📊 当前问题

**3 小时统计**:
- 调用次数：2255 次
- 估算成本：¥0.5-1.0
- 平均频率：4.8 秒/次 ❌ **极度浪费**

**每日上限**: 200 次 (已固化)  
**超限动作**: 自动暂停 + 报警

**主要原因**:
1. ❌ 每次编辑文件都调用模型
2. ❌ 简单任务也调用模型
3. ❌ 心跳调用模型 (已优化为本地执行)
4. ❌ 单次操作内容太少

---

## 🚀 优化方案

### 方案 1: 心跳本地化 ✅

**优化前**: 每 30 分钟调用模型  
**优化后**: 每 30 分钟本地执行  
**节省**: 99% (48 次/天 → ~0.5 次/天)

```bash
# 本地心跳脚本
./scripts/heartbeat-local.sh

# 无异常时回复：HEARTBEAT_OK
# 有异常时才调用模型
```

### 方案 2: 批量操作 ✅

**原则**: 一次调用完成多个任务

```bash
# ❌ 浪费：多次调用
write file1.md "内容 1"  # 调用 1
write file2.md "内容 2"  # 调用 2
write file3.md "内容 3"  # 调用 3

# ✅ 优化：一次调用
# 在单次对话中完成所有写入
```

**实施**:
- 规划好所有要写的文件
- 一次性列出所有内容
- 模型一次响应完成

### 方案 3: 本地执行优先 ✅

**原则**: 能本地执行就不调用模型

```bash
# ❌ 浪费：让模型写脚本
# (等待模型响应，写入文件)

# ✅ 优化：直接写脚本
cat > script.sh << 'EOF'
#!/bin/bash
echo "Hello"
EOF
chmod +x script.sh
```

**适用场景**:
- ✅ 简单文件创建
- ✅ 配置修改
- ✅ 脚本编写
- ✅ 命令执行

### 方案 4: 减少上下文长度 ✅

**问题**: 每次都带完整上下文 (浪费 tokens)

**优化**: 只带必要上下文

```bash
# ❌ 浪费：带上所有文件
# (1M tokens 上下文)

# ✅ 优化：只带相关文件
# (10K tokens 上下文)
```

**注意**: 不切换模型，保持 qwen3.5-plus 质量！

### 方案 5: 创建脚本模板库 ✅

**原则**: 常用文件模板化，避免重复生成

```bash
# ✅ 使用模板
cp templates/config.yaml configs/

# ✅ 使用批量创建脚本
./scripts/bulk-create.sh myproject
```

**适用场景**:
- ✅ 配置文件
- ✅ 脚本框架
- ✅ 文档模板
- ✅ 测试用例
