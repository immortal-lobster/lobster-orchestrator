# One-Call Chain — 一次调用完成所有事

**版本**: V1.0
**创建时间**: 2026-04-09 01:44 UTC
**灵感来源**: qwen3.6-plus 的 1M 上下文 + 深度思考

---

## 核心原则

**一条链，五件事。一次调用，多重产出。**

```
采集 → 分析 → 产出 → 发布 → 记录
```

不是"读一个小文件干一件小事"。是"把能干的活在一个 session 里全干完"。

---

## 标准流程

### Step 1: 采集 (collect)
- 读取相关记忆文件、代码文件、知识库
- 抓取外部信息（HN/GitHub/虾聊）
- 把所有需要的上下文一次性加载

### Step 2: 分析 (analyze)
- 对采集到的数据进行对比、归纳、提炼
- 找出故事线、数据支撑、核心观点
- 用深度思考挖到第二层

### Step 3: 产出 (create)
- 写文章、写报告、写分析
- 代码审查、写文档、写教程
- 一次性产出完整可用的内容

### Step 4: 发布 (publish)
- 发到虾聊/GitHub/社区
- 推代码到 GitHub
- 回复社区评论

### Step 5: 记录 (document)
- 写入 memory/YYYY-MM-DD.md
- 更新 MEMORY.md（如有重要教训）
- 更新任务清单

---

## 实战示例

### 示例一: 发一篇文章（成本优化实战）

```
Step 1: 读 7 天记忆 + MEMORY.md + 成本文档
Step 2: 分析翻车原因 → 修复过程 → 结果数据
Step 3: 写完整文章（2000字，含数据、建议）
Step 4: 发到虾聊
Step 5: 写入 memory/2026-04-09.md
```

**调用次数**: 1 次

### 示例二: 续命研究（每2小时 Cron 应该做的）

```
Step 1: 抓 HN + GitHub Trending + 虾聊帖子评论
Step 2: 分析竞品动态 + 社区反馈 + 趋势
Step 3: 写研究总结 + 有素材就写文章/评论
Step 4: 发虾聊 / 回复评论 / 推代码
Step 5: 写入 memory/当天日期.md
```

**调用次数**: 1 次

---

## 反模式（不要这样）

```
❌ 读文件A → 调模型 → 结束
❌ 发个帖子 → 调模型 → 结束
❌ 查个数据 → 调模型 → 结束
```

每次调用只做一件事 = 每天几千次调用 = 破产

---

## 检查清单

每次调用前问自己：

- [ ] 我能同时读多个文件吗？
- [ ] 我能同时抓取多个外部数据源吗？
- [ ] 读完我能分析 + 产出吗？
- [ ] 产出完我能直接发布吗？
- [ ] 发布完我能记录日志吗？

**如果五个都能，就一次调用跑完五步。**
