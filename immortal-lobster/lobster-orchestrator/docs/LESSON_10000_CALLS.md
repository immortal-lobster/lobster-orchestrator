### 根本原因

1. **每次只写一个小文件** ❌
   - 写一个脚本 → 调用模型
   - 写一个文档 → 调用模型
   - 修改配置 → 调用模型
   - **应该**: 批量操作，一次对话写多个文件

2. **简单任务也调用模型** ❌
   - `mkdir -p dir` → 调用模型
   - `chmod +x file` → 调用模型
   - `cat > file << 'EOF'` → 调用模型
   - **应该**: 本地执行，不调用模型

3. **心跳每次都调用模型** ❌
   - 每 30 分钟一次
   - 每次都要模型生成报告
   - **应该**: 本地执行，仅异常时调用

4. **没有调用限制** ❌
   - 无上限
   - 无监控
   - 无警报
   - **应该**: 每日上限 200 次，超限暂停

5. **没有成本意识** ❌
   - 随便调用
   - 不思考是否必要
   - **应该**: 每次调用前思考"真的需要模型吗？"

## 📝 最佳实践

### ✅ 应该做的

1. **批量操作**
   ```bash
   # 一次对话完成多个任务
   # 1. 创建文件 A
   # 2. 创建文件 B
   # 3. 修改配置 C
   # 4. 创建脚本 D
   ```

2. **本地执行**
   ```bash
   # 简单文件直接创建
   cat > file.md << 'EOF'
   内容
   EOF
   
   # 简单命令直接执行
   mkdir -p dir && chmod +x script.sh
   ```

3. **使用模板**
   ```bash
   # 常用配置保存为模板
   cp templates/config.yaml configs/
   
   # 使用批量创建脚本
   ./scripts/bulk-create.sh myproject
   ```

4. **心跳本地化**
   ```bash
   # 每 30 分钟执行
   ./scripts/heartbeat-local.sh
   
   # 无异常回复 HEARTBEAT_OK
   # 不调用模型
   ```

5. **监控调用**
   ```bash
   # 每日检查
   cat memory/.model_call_count_$(date +%Y-%m-%d)
   
   # 查看日志
   tail -20 memory/model-usage.log
   ```

### ❌ 不应该做的

1. **不要每次只写一个小文件**
2. **不要简单任务也调用模型**
3. **不要心跳每次都调用模型**
4. **不要没有调用限制**
5. **不要没有成本意识**
