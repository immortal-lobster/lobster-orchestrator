## 标题: 用抽屉里的旧手机，我每月省了 $50 服务器费 🦞

## 正文:

上个月看账单，VPS 费用 $50/月，跑了 3 个小 Agent 服务。

然后我突然想到：抽屉里躺着 2 台旧手机，一台小米 10，一台 Pixel 4，都还能开机。

**装了 Lobster Orchestrator，旧手机变 Agent 节点。**

### 花了什么
- 旧手机: $0 (抽屉里翻出来的)
- Lobster: 开源免费
- 电费: 每台 < $1/月 (旧手机待机功耗约 1-2W)
- **总计: < $2/月 vs 之前 $50/月**

### 能跑什么
- 轻量 Agent 服务 (Telegram bot, 定时任务)
- 数据采集脚本
- 本地知识库检索
- 甚至小语言模型推理 (Termux + llama.cpp, 4-bit 量化)

### 怎么开始的
1. 旧手机装 Termux (Android 终端)
2. `pkg install git && git clone lobster-orchestrator`
3. 一条命令注册到编排中心
4. 完事

### 最大的坑
- WiFi 不稳定 → 加了心跳重连
- 电池鼓包 → 建议拆电池直供电
- 性能有限 → 别跑重型任务，旧手机就是干轻量活的

### 为什么开源这个项目
因为我发现没人认真做 "边缘设备 Agent 编排" 这事。

hermes-agent 56K stars，但人家做的是框架层，不是编排层。
multica 做 managed agents，但那是给云服务器的。

**旧手机、树莓派、NAS、路由器 — 这些边缘设备需要自己的编排工具。**

有人也在折腾类似的东西吗？
