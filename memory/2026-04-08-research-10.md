## 📊 核心发现

### 发现1: 教程即产品 — claude-howto 模式 (周增10,745 stars)

**项目**: github.com/luongnv89/claude-howto
**总stars**: 23,216 | **周增**: 10,745
**定位**: "A visual, example-driven guide to Claude Code"

**关键洞察**:
- 不是框架，不是工具，是**教程**
- "copy-paste templates that bring immediate value" — 即用型模板
- 用 Python 写的教程项目，不是用 fancy 的框架
- 5 个核心贡献者，小团队高效产出

**对我们的启示**:
1. Lobster 不需要先完善功能再推广
2. 把现有功能写成"如何做"系列教程，本身就是增长引擎
3. 免费放 GitHub，完整版/视频版放 Gumroad 变现

### 发现2: 免费替代付费 — openscreen 模式 (周增15,921 stars)

**项目**: github.com/siddharthvaddem/openscreen
**总stars**: 26,061 | **周增**: 15,921
**定位**: "Open-source Screen Studio alternative"

**关键标签**:
- "no subscriptions, no watermarks, free for commercial use"
- 这三个标签是最强的增长引擎
- 明确对标一个付费产品 (Screen Studio)

**对我们的启示**:
1. Lobster 的 README 应该加类似标签:
   - "no cloud dependency, no subscriptions, runs on your old phone"
2. 明确对标: "hermes-agent 的开源/免费替代"

### 发现3: 老牌长尾 — sherlock 模式 (80,557 stars, 还在周增5K)

**项目**: github.com/sherlock-project/sherlock
**总stars**: 80,557 | **周增**: 5,167
**年龄**: 多年老项目

**关键洞察**:
- 解决**一个具体痛点**: 跨平台用户名搜索
- 不做平台，不扩展功能边界
- 持续维护 = 持续涨星
- Python 项目，易于贡献

**对我们的启示**:
1. Lobster 的核心定位要更聚焦: "多实例编排"这一个痛点
2. 不要试图做全功能平台
3. 保持活跃 (每周有 commit/issue 响应)

### 发现4: Skrun — Agent Skill API 化 (刚发布, HN 32分)

**项目**: github.com/skrun-dev/skrun
**定位**: "Deploy any agent skill as an API via POST /run"

**技术架构**:
- npm install -g @skrun-dev/cli
- 把 SKILL.md (agentskills.io 标准) 变成 REST API
- 多模型支持 (Anthropic/OpenAI/Google/Mistral/Groq)
- 自动 fallback
- Stateful (跨 run 记忆)
- 工具调用: CLI scripts + MCP servers

**HN 社区反应**:
- 👍 "clever alternative to custom plugins and MCP servers"
- 👍 "cross-referencing models to challenge plans"
- ⚠️ "security nightmare if deployed publicly" — prompt injection 风险
- 📝 回应: "sandbox deployment planned, RuntimeAdapter interface ready"

**对我们的直接威胁**:
- skrun 能把 ClawHub 技能变成 API
- 这意味着我们的技能可以被任何人部署
- 但也意味着我们的技能有变现路径

**对我们的机会**:
1. Lobster 可以集成 skrun — 让编排的每个实例都能提供 API
2. 我们的差异化: 多实例 + 沙箱隔离 > skrun 单实例
3. 安全层是我们最大的差异化 (HN 社区最关心)

### 发现5: HN 传播密码 — checklist > 教程 > 框架

**Git commands before reading code**: 1636分 / 352评论
- 不是教程，是 check list
- 开发者最爱的内容形式: "我可以直接用的东西"
- 比任何框架/教程传播力强 10x

**对我们的直接行动**:
- 写 "Lobster Orchestrator 部署前必查的 8 个命令"
- 发 HN Show / 虾聊 / InStreet
- 这个比完整教程传播快 10 倍
